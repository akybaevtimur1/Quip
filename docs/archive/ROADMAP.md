## R1 — Reframe 2.0 (✅ СДЕЛАНО 2026-06-08, коммиты 352284c→d234b92)
Проблема была: «флеши» = смена кропа не попадает точно в склейку камеры; перебивки/b-roll
(без лица) кропались узким слайсом (трава вместо «показать широко»). Корень: грубый ffmpeg
scene-порог (то ложные, то пропуски) + кроп одним «выражением во времени» (промах на кадр).
**Решено per-shot моделью (как Opus):**
- **Точные склейки:** PySceneDetect ContentDetector (frame-accurate) вместо ffmpeg-порога —
  на comedy01 clip_01 14 кадроточных склеек vs 8 у ffmpeg@0.4. ⚠️ scenedetect НЕ был в депах
  (доки врали) — добавлен в БАЗОВЫЕ. Шкала порога ~27 (НЕ ffmpeg 0..1) → config reframe_scene_threshold.
- **Рендер ПО-ШОТНО (сегменты + concat):** каждый план — отдельный ffmpeg-сегмент со своим
  static-кропом → concat встык → субтитры ВТОРЫМ проходом. Кроп меняется ровно на склейке
  by construction (промахнуться на кадр нельзя). merge_shot_plan сливает равные планы (статика → 1 кодировка).
- **Режим НА ШОТ:** лицо → fill (центр=медиана/ASD говорящий); нет лица → fit (весь кадр + блюр,
  b-roll широко). build_shot_plan. (Удалён time-expr build_vf_dynamic — источник промаха на кадр.)
- DoD ✅: comedy01 (5 валидных клипов, clip_01 5 fit-шотов из 11) + Comedians-in-Cars 300–330с
  (13 склеек): fit-перебивки широко, reframe следит за говорящим, граница t=21.655 чистая (нет флеша).
  Скрины отправлены фаундеру. ⚠️ Comedians целиком через UI не прогнан (Deepgram 408 SLOW_UPLOAD —
  upload 37МБ wav, сетевое; reframe тестился напрямую по видео).
- Возможные тюнинги (по фидбеку): scene_threshold/min_scene_sec; короткие fit-перебивки (детект-
  промах лица на быстром плане → ложный fit); масштаб fill-кропа; speaker-путь ещё на ffmpeg detect_cuts.

## R2 — Не только подкасты + мягкий empty-state
Сейчас stage2 (Gemini) заточен на «реплики одного спикера» → на не-речевых видео (влог,
геймплей, музыка, экшен) возвращает 0 → «интересных моментов не нашлось» (фаундер: бесит).
- Промпт выбора моментов расширить: визуальные хуки/экшен/смена темы, а не только речь.
- Empty-state мягче + диагностика «почему» (язык/мало речи/тип контента), не глухое «ничего нет».
- DoD: не-подкаст видео даёт клипы; «нашлось 0» только когда реально пусто.

## R3 — Стили субтитров (выбор)
