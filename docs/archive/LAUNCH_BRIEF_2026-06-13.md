## 0. Решения этой сессии (зафиксировано, НЕ переобсуждать)

### Инфраструктура лаунча
| Слой | Где | Стоимость | Статус |
|---|---|---|---|
| Фронт | **Vercel** (Hobby) | free | есть код, нужен деплой (фаундер) |
| Бэк: API + компьют | **Modal** (serverless, scale-to-zero) | free $30/мес кредитов | `deploy/modal/` спайк работает; render-portability фикс открыт |
| Auth + DB + Storage + Realtime | **Supabase** (**Pro**) | **$25/мес ← сюда $50 гранта Chevron** | НЕ начато (нужны аккаунт/секреты фаундера) |
| Хранилище клипов | Supabase Storage → Cloudflare R2 потом | в Pro | при росте egress → R2 (egress $0) |
| Оплата | **Lemon Squeezy** (Merchant-of-Record, сам считает налоги) | % с продаж | НЕ начато |
| Транскрипция | **Deepgram** Nova | ~$0.0043/мин ($200 free на старте) | работает |
| LLM (выбор моментов/главы/хуки) | **Gemini** `gemini-flash-latest` | копейки PAYG | работает. ⚠️ НЕ ставить `gemini-2.5-pro` (квота 0 на free) |

**Почему деньги гранта именно в Supabase Pro:** грант = «фикс-подписка с чеком, нет
free-альтернативы». Под это правило проходит ТОЛЬКО Supabase (free пауза через 7 дней
простоя + нет бэкапов — недопустимо для живого платного продукта). Всё остальное —
free/PAYG → грант отсёк бы сам.

**Архитектура (почему страница не тупит):** фронт читает статус/клипы НАПРЯМУЮ из
Supabase (always-on); Modal просыпается только на реальную работу (cold-start прячется
за минутным пайплайном). Поллинг бьёт в Postgres, не в тяжёлый воркер.

### Открытая безопасность
- В истории GitHub (коммит `59d07b4`) лежали YouTube-куки → **фаундер перевыпускает**
  (logout/login + новый cookies.txt). Действие фаундера, не агента.

## 1. Позиционирование (наш wedge против Vizard/Opus)

Vizard = «30+ клипов в один клик» → максимизируют **output**, разрослись в комбайн
(scheduling, teams, AI Studio, UGC-ads, API, mobile). Их слабости (из отзывов G2/
Trustpilot/SoftwareAdvice):
1. **AI-выбор мажет** — юзеры руками правят тайминги/моменты (#1 жалоба).
2. **Мутные кредиты + surprise-paywall + refund-войны** (#2 жалоба).
3. **Overbuilt** — соло-креатору слишком много.

**ClipFlow = «меньше клипов, но ты понимаешь ПОЧЕМУ их стоит постить».** Максимизируем
**confidence**, не output. Мы УЖЕ владеем сложной половиной, которую они не закрыли:
- **reason + score + type на клип** (объяснимость) + **reframe без флешей Δ=0**.
- Простой честный прайсинг без казино.

Лендинг-мессадж (фаундеру на дизайн): **«Don't just get clips. Know why they're worth
posting.»**

### НЕ строим (размывает wedge, сожрёт неделю)
Scheduling/календарь · команды/воркспейсы · AI Studio/text-to-video/UGC-ads · мобильное
приложение · сложные кредиты с rollover. (GPT-анализ согласен.)

## 2. Бизнес-модель / прайсинг (простой, анти-Vizard)

Единица стоимости = **минуты исходника** (доминанта затрат — транскрипция, растёт с
длиной источника). Но подаём ПРОСТО, без кредитов-казино:

| План | Цена | Лимит | Фичи |
|---|---|---|---|
| Free | $0 | 1-2 видео/мес, watermark, 720p | честный preview, без surprise-paywall |
| Starter | ~$10-15/мес | N минут/мес или X видео | без watermark, 1080p, все стили |
| Pro | ~$25-30/мес | больше минут | приоритет, soon-фичи |
| Founder pass | — | ранним юзерам | привлечение первых людей |

Правила (учесть боли Vizard): **никакого surprise-paywall** (заранее показать что free
даёт), **простые лимиты** (X видео/минут, не кредиты), честный бесплатный preview.

Маржа (замер этой сессии): видео ~6 мин CPU-компьюта (reframe-ASD + render), Deepgram
~$0.0043/мин. На Starter $10/10 видео ≈ $1/видео при cost ~$0.24 (Deepgram 30мин) →
маржа ~76%. На Modal компьют ~$0.02/клип. Юнит-экономика здоровая.

## 4. SCOPE НОЧНОЙ СЕССИИ (ordered, top = делать первым)

Каждая задача — по правилам `CLAUDE.md`: **тест-первым на pure-логике, `just check`
зелёный перед коммитом, conventional commits, без тихих фолбэков, без скоупа сверх
списка**. Дизайн НЕ трогать. Аккаунты/секреты НЕ создавать.

### T1 — Хук / топ-текст (ФЛАГМАН, делать первым)
Gemini генерит короткий цепляющий хук-заголовок на клип, **привязанный к reason**
(объяснимый — наш отличитель). Хранение в модели (`Clip.hook` или top-overlay в
`CaptionTrack`). Рендер — отдельный ASS top-event (позиция сверху, своя стилизация, НЕ
пересекается с нижними субтитрами, кадр 9:16). Правится в редакторе (новый таб/поле).
Опция вкл/выкл и «первые N секунд / весь клип».
**DoD:** клип имеет hook; в mp4 сверху горит заголовок; правится в UI и перерендеривается;
TDD на pure-билдере top-event (позиция/перенос/тайминг); `just check` зелёный.
**OSS/доки:** топ-текст рендерим тем же стеком, что субтитры (ASS top-event +
libass-превью) — НЕ изобретать второй пайплайн. ⚠️ Шрифты класть в ОБА места
(`apps/web/public/libass/fonts` + `services/worker/fonts`), libass `fallbackFont`
обязателен (иначе тихий CSS-фолбэк — см. HANDOFF §0 / память libass-preview-gotchas).
Анимацию хука (если делаешь) — глянуть готовое (ASS `\t` уже в проекте; Revideo-спайк
рядом как референс), не хендроллить слабое.

### T2 — Богатый reasoning как продукт
Расширить Gemini-схему сегмента: `hook_line`, `why_works`, `clip_kind`
(hook|emotion|story|quote|insight). Показать в `ClipCard` структурно (не один blob).
**DoD:** сегменты содержат поля; ClipCard рендерит; тесты на постобработку/валидацию;
`just types` прогнан (контракт менять ТОЛЬКО в models.py).

### T3 — Сочные субтитры: keyword-highlight + эмодзи
Авто-подсветка ключевых слов (расширить существующие `emphasis_color`/`emph_positions`)
+ опц. эмодзи в репликах (флаг). Не ломать инвариант WYSIWYG (превью libass = экспорт).
**DoD:** ключевые слова окрашены в ЭКСПОРТЕ (не только CSS-превью); тесты; превью совпадает.

### T4 — Баги из HANDOFF §0.1
Приоритет: **#4** (hl.box/scale в ASS-экспорте — «слово в плашке» обещано пресетами, в
libass только цвет) · **#8** (двойные субтитры — детект вшитых, пропуск stage4) · **#2**
(превью-кадр устаревает после драга шортса — лёгкий ре-анализ интервала) · **#9** (кнопка
«Повторить» для глав на free-tier).
**DoD:** на каждый — доказательство (скрин/вывод), тест где применимо.

### T5 — Соотношения сторон (parity)
9:16 + 1:1 + 4:5 + 16:9. Config + render + UI-тоггл в редакторе. Reframe-геометрия под
каждый ratio.
**⛔ ПЕРЕД ЭТОЙ ЗАДАЧЕЙ — читай `docs/REFRAME_FPS_GRID_INVARIANT.md`.** T5 трогает
render/reframe-геометрию → сломаешь кадровую сетку = вернутся «флеши» (видно ТОЛЬКО на
видео ≠ 25 fps, тесты будут зелёные). Сохрани целочисленную сетку кадров.
**DoD:** рендер в каждом ratio (ffprobe размеры); тест на расчёт кропа; fps-grid Δ=0
сохранён; `just check`.

### T6 — Прайсинг/лимиты модель + Supabase-ready (БЕЗ секретов)
Простая модель лимитов (X видео/минут/мес, не кредиты). Схема БД (users/jobs/usage) +
SQL-миграции под Supabase Postgres + адаптер поверх `db.py` (сейчас SQLite). **НЕ создавать
Supabase-проект, НЕ трогать секреты** — написать КОД и миграции, готовые к подключению;
задокументировать что фаундер должен вписать (URL/anon key/service key).
**DoD:** схема + миграции написаны; db-адаптер с тестами (на SQLite-режиме); документация
«что вписать фаундеру».

## 4.5 Принципы работы (ВСЕГДА — не только ночь)

- **Не изобретай велосипед. OSS-first.** Перед тем как писать своё — ищи готовое
  open-source решение/код, проверяй лицензию (MIT/Apache/BSD → можно вендорить), бери
  проверенное. Стабильность важнее самописного. (Так пришли libass, PySceneDetect,
  LR-ASD, Revideo — НЕ хендроллить слабые версии того, что уже решено.)
- **Сверяйся с АКТУАЛЬНОЙ документацией** (context7 / официальные доки), не с памятью —
  особенно Next 16 (ломает обучающие данные), Gemini SDK (`google-genai`), ffmpeg-фильтры,
  libass/ASS-теги, Supabase.
- **Используй агентов**, когда реально удобно (широкий fan-out поиск по кодовой базе;
  параллельные независимые куски). Не ради галочки.
- **Скиллы** можно добавлять/удалять под задачу (неиспользуемые — убирать).
- **🎨 ДИЗАЙН НЕ ДОЛЖЕН ХРОМАТЬ.** Да, фаундер перепишет визуал завтра — НО: плохой UI →
  труднее понять что работает И труднее переписывать; UI/UX — огромная часть ценности
  продукта. Любой новый UI (поле хука, тоггл ratio, показ reasoning) делай **когерентно к
  существующему**: палитра Warm Charcoal+Coral (`apps/web/app/globals.css` `@theme`),
  паттерны существующих компонентов редактора/табов (`components/editor/*`), шрифты
  Unbounded/Onest. Чисто и юзабельно. Не пиксель-перфект, но и НЕ мусор.
- **Документируй ВСЁ** по ходу: журнал `CLAUDE.md` + `HANDOFF.md` §0 + финальный отчёт.
