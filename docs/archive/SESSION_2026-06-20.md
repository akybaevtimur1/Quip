## Part 2 — Render quality fix (Pro looked bad)

- **Root cause:** the final clip encode was hardcoded `-preset veryfast -crf 20` for **every plan**
  in all three render paths (`build_single_pass_cmd`, `render_frame_by_frame`, `build_timeline_cmd`).
  Pro got the same soft encode as Free, just 1080 vs 720.
- **Fix:** encode quality is now part of the server-side `RenderPolicy` (un-spoofable): added
  `PlanLimits.video_crf`/`video_preset` → free `20/veryfast`, **paid `18/medium`** (crisper at the
  same 1080). Threaded through `render_clip` + `render_timeline` via a shared `_video_out_args`.
  Encode-only — trim/fps/regions untouched, reframe Δ=0 intact. TDD on `resolve_render_policy`.
- Commit `b6d72b2`. Deployed (`modal deploy`). Caps stay 1080×1920 (practical Shorts max); can bump
  paid to `17/slow` for absolute max if desired.

## Part 3 — Live Clip Feed (kill the dead pre-render wait)

**Problem:** the user only saw anything about their video *after* render; the first ~60% was a static
stepper. But the pipeline already computes the valuable part (each clip's hook / "why it's worth
posting" / score / interval) at the `select` stage — it was just gated behind the render boundary.

**Research (2 Opus sub-agents, real web research):** no competitor (OpusClip, Vizard, Klap, Munch,
2Short, Quso, Submagic, Descript…) streams pre-render clip info or shows "why this clip" during the
wait — all batch at the end; only Vizard narrates stages (labels only). So surfacing our data early
*leapfrogs* the field. Spec `docs/superpowers/specs/2026-06-20-live-clip-feed-design.md`, plan
`docs/superpowers/plans/2026-06-20-live-clip-feed.md`.

**Built (Variant A):**
- **W1 (core):** `set_clips_pending` now runs **right after `select`** (status `selecting`,
  progress 60) instead of the render boundary → the frontend (which already swaps the stepper for the
  grid on clip-presence) shows rich cards minutes earlier. Order preserved (idx aligned; regression
  test). `set_clips_pending` gained a `status` param (cloud + sqlite).
- **F2/F3 (Opus sub-agent):** `PendingThumb` client frame-grab from the preview proxy fills the
  "Rendering…" box; cards pop in (fade+scale) with the score counting up (reduced-motion respected).
- **Thumbnail CORS fix:** live check showed the frame-grab video was CORS-blocked (preview.mp4
  302→`cdn.quip.ink` with no ACAO header + `crossOrigin="anonymous"`). We only *display* the canvas
  (never read pixels), so dropping `crossOrigin` fixes it (tainted canvas still paints). Commit
  `75c484b`.
- **W2/F1 + migration 0011:** `jobs.source_minutes/transcript_words/moments_found` (nullable,
  additive — applied to prod Supabase + local SQLite). `db.set_progress_detail` (best-effort, logs on
  failure — cosmetic, never fails a paid render). `JobProgress` shows "· 8.3 min / · 412 words /
  · 9 found" in the 0–60% window. Commit `5c4f389`.
- Commits: `c8509e1` (spec), `0a1d831` (plan), `d3d4f7f` (W1+F2/F3), `75c484b` (CORS), `5c4f389`
  (W2/F1), `67d6117` (docs sync).

**Verified live (real upload):** ~30s after submit, 4 rich cards at "0 of 4 ready" with hook
(«Почему ты не помнишь свою жизнь»), WHY IT WORKS, score 0.92 — **before any render** — streaming to
"1 of 4 ready". Core confirmed. (The narration counts weren't re-eyeballed because that run hit the
transcription cache, so its 0–60% window was too fast; they're shipped + gated.)

## Part 4 — Live moment discovery / co-watch (SHIPPED to `main`, deployed)

**Why:** the founder found the early-window narration too text-y and wanted a **visual "co-watch":**
their source video playing immediately during processing with detected moments surfacing as they're found.

**Key decision (founder's gate):** does live moment-detection influence the AI's clip *quality*?
**No — cosmetic by design.** Markers come from a separate pure heuristic that is **never fed into the
Gemini `select_segments` call** — the authoritative selection (quality core) is untouched, final clips
identical. So (per the rule "if it doesn't influence quality, do the harder live version") we built the
true live variant in a branch, verified the visual, then merged.

**Built (worker, all TDD, ⚠️ never feeds `select_segments`):**
- `app/pipeline/preview_moments.py` (pure): `detect_preview_moments(words)` — language-neutral signals
  (question `?`, emphasis `!`, stat digit, beat = pause) each carrying the **real transcript phrase**
  (`PreviewMoment.text`); `extract_loudness` (ffmpeg→numpy RMS dBFS, **best-effort**, [] on any failure)
  + `detect_energy_moments` (pure peak picker) so markers appear **early** (after download, during the
  long transcribe wait); `merge_moments`. 14 unit tests.
- `PreviewMoment` model (`models.py`); persisted to `job_artifacts.preview_moments` (**migration 0012**,
  applied to prod) via `db.put_job_artifact` (upsert) in `run.py` — energy markers after download,
  merged with transcript markers after transcribe. `GET /jobs/{id}/preview-moments` serves them.
- **Invariant enforced + commented:** `run.py` computes markers but passes only `transcript` to
  `select_segments`; clip quality unaffected.

**Built (frontend):**
- `CoWatch.tsx` — **v2 design** after founder feedback ("the timeline bar reads as neiroslop; eye
  ping-pongs point↔line"). Killed the abstract bar; now the AI surfaces the **real line it caught as a
  quote chip ON the video** (phrase + tag: Question / Big moment / Number / Beat), rising in as moments
  arrive. Eye stays on the content; markers carry meaning. (`riseIn` keyframe in `globals.css`.)
- `CoWatchPanel.tsx` — polls `getPreviewMoments` and **reveals markers one at a time** (gentle stream
  regardless of backend batching — the "labor illusion" with the user's own content); keeps the
  free-phase **Stop** control.
- Dashboard wiring (`dashboard/page.tsx`): on upload, stash `URL.createObjectURL(file)` → the co-watch
  plays the user's **own file instantly** (no upload round-trip / CORS). Shows during `tracking`;
  **graceful handoff** to the real clip grid the instant clips exist (`showProgressiveGrid`). YouTube /
  post-reload → falls back to the stepper.
- `/dev/cowatch` harness (unauthed) for visual review without a full pipeline run.

**Why it's useful (not theater), the design principles that drove it:** proof-of-work → trust (real
lines prove the AI read *their* video); the labor illusion (visible work shrinks perceived wait);
immediacy (their own file, now); continuity (moments preview where clips will come from). The handoff
to the real AI-selected clips keeps it honest — the chips are cosmetic attention, not the deliverable.

**Verified:** the co-watch visual via the `/dev/cowatch` harness (screenshots). API live-verified on
prod (`/preview-moments` returns `{moments:[]}` for an unknown job = endpoint deployed). Full live
upload-flow eyeballing left to the founder ("test in real life").

## Part 5 — Hook size (and ALL re-render edits) not applied on download — CDN edge-cache

**Symptom (founder):** changing a clip's hook **size** in the editor, then Render → Download, showed no
change — even at extreme sizes.

**Root cause (Opus sub-agent, isolated worktree, systematic-debugging — traced, not guessed):** the
hook size **is** carried correctly to the ASS end-to-end. The bug is **CDN edge-cache staleness**:
`storage.upload_clip` re-renders into the **same R2 key** (`{job}/{clip}_captioned.mp4`, overwrite) and
returns a stable `cdn.quip.ink` URL, but the upload set **no `Cache-Control`** → Cloudflare served the
**stale first render** on re-download. Hits **every** edit after the first render; hook size was just the
most visible.

**Fix (defense-in-depth, export/cache only — frame-grid untouched):** (1) `Cache-Control: no-cache` on
clip upload (`clip_upload_extra_args`); (2) `?v=<clip_edit.version>` cache-buster on the render URL
(`storage.with_cache_bust`, applied in `get_render`; version bumps every PATCH → guaranteed CDN miss
even under a Cloudflare "cache everything" rule). 13 storage tests. Commit `fa14a5c` on `main`, worker
deployed. JOURNAL entry added.

## Key learnings captured
- **Live authed editor on Windows:** offline cookie import is dead (Chrome App-Bound Encryption);
  use the Playwright MCP persistent browser + manual quip.ink login. (`memory/live-editor-browser-access.md`)
- **React Compiler lint** (`react-hooks/refs`, `react-hooks/immutability`, `set-state-in-effect`):
  don't read a ref's `.current` into a hook/state-setter arg; attach DOM listeners in effects /
  callback refs; seed reduced-motion finals via `useState` initializers.
- **Video frame-grab + CDN:** `crossOrigin="anonymous"` needs CORS headers (the CDN lacks them);
  drop it and display the tainted canvas (no pixel readback).
- **Persist-at-select** is safe because the render fan-out reuses the same `segments` order
  (clip_{i:02d} idx aligned).
- **Commit gate:** `just check` green before every commit; commit from PowerShell (pre-commit hook
  needs `just` on the refreshed PATH); Cyrillic messages via `git commit -F <utf8 file>`.
