# WYSIWYG-превью редактора: libass + ffmpeg (Approach A)

> Статус: УТВЕРЖДЁН фаундером 2026-06-12. Приоритет — ОПЕН-СОРС (MIT), «вижу изменения сразу».
> Не трогаем ffmpeg-рендер и защищённый инвариант reframe (`docs/REFRAME_FPS_GRID_INVARIANT.md`).

## 1. Цель
В модалке-редакторе пользователь должен **сразу видеть изменения**:
- субтитры (текст/стиль/позиция/пресет) — **пиксель-в-пиксель как на финальном экспорте**;
- перемещение/растяжение шортса на таймлайне — **видео реально едет на новый момент**.
Всё на чистом OSS (MIT), без лицензионных ловушек, без замены рендера.

## 2. Ключевая идея (почему именно так)
Финальный экспорт жжёт субтитры через **ASS** (`captions_v2.compile_ass` → libass внутри ffmpeg).
**SubtitlesOctopus / libass.wasm (MIT)** — это ТОТ ЖЕ libass, скомпилированный в WASM, рисующий ТОТ
ЖЕ ASS в браузере. → Один источник правды (ASS), один движок (libass) → **превью субтитров =
экспорт субтитров точь-в-точь.** Это снимает главную боль «не вижу, как будет», без Remotion и
без переписывания рендера.

## 3. Решения (зафиксированы)
- **Превью-видео = `source.mp4`**, перемотанный на текущий интервал клипа (луп), показан в 9:16-
  контейнере с CSS-кропом по центру reframe-региона. Двигаешь блок на таймлайне → seek → видео едет.
- **Субтитры = libass.wasm** поверх 9:16-контейнера; ASS берём с нового бэкенд-эндпоинта (компиляция
  текущего edit-state). Время гоним вручную: `octopus.setCurrentTime(video.currentTime − source_start)`
  (ASS — в клип-времени, видео — в source-времени).
- **Правка субтитров на видео**: libass рисует в `<canvas>` (по словам не кликается) → кликабельная
  ОДНА хит-зона на текущую активную реплику → inline-textarea → PATCH → перезапрос ASS → libass
  обновляется живьём. Стиль/пресет/позиция → перекомпиляция ASS → мгновенно.
- **Экспорт — ffmpeg, без изменений.** Инвариант reframe не трогаем.
- **Кроп в превью — CSS-приближение** (MVP: центр первого региона; динамика по шотам — рефайнмент).
  Субтитры — точные. Точный live-кроп = Approach B (ниже).

## 4. Компоненты (изолированные единицы)
1. **Бэкенд `GET /jobs/{job}/clips/{clip}/ass`** → `text/plain` ASS. Берёт edit-state (store.load_edit
   или ensure_edit), `store.load_transcript_words`, `ClipTimeMap(edit.source_intervals)`,
   `captions_v2.compile_ass(edit.captions, words, cmap)` → возвращает строку. 404 если нет клипа.
2. **api.ts `getClipAss(jobId, clipId)`** → `Promise<string>` (fetch text).
3. **Ассеты libass** в `apps/web/public/libass/`: воркер+wasm из npm `libass-wasm`, шрифт Montserrat.
4. **Фронт `LibassLayer.tsx`** — обёртка SubtitlesOctopus. Пропсы:
   `{ videoRef, assText, timeOffsetSec, className? }`. Создаёт инстанс на videoRef + subContent=assText,
   workerUrl=`/libass/subtitles-octopus-worker.js`, fonts=[Montserrat], ручной `setCurrentTime` по
   rAF из `video.currentTime − timeOffsetSec`. Пересоздаёт/`setTrack` при смене assText. Чистит в unmount.
5. **`ClipEditorModal` превью-рерабоут**: `<video src=source>` (seek на интервал, луп), 9:16 CSS-кроп
   из регионов, `<LibassLayer>` поверх, хит-зона клика на активную реплику → правка → recompile ASS.
   Таймлайн/вырез слов — как есть; на изменение интервала — reseek превью.

## 5. Что НЕ трогаем
ffmpeg stage5 + reframe-инвариант; бэкенд-пайплайн (download/transcribe/select/analyze); таймлайн-
drag/resize; вырезание слов. Грид-карточка может оставить лёгкий CSS-`CaptionOverlay` (libass — только
в модалке, где нужна точность).

## 6. Обработка ошибок
- Бэкенд: нет клипа/edit → 404 (как у `/analysis`).
- Фронт: ASS-fetch упал → тост, превью без субтитров (не молча). libass-инициализация упала →
  фолбэк на старый CSS-`CaptionOverlay` (деградация, не белый экран).

## 7. Тестирование
- Бэкенд: unit (TestClient) — `GET .../ass` возвращает непустой валидный ASS (есть `[Script Info]`,
  `[Events]`), 404 на несуществующий клип.
- Фронт: `just check` (tsc+eslint) зелёный. ⚠️ Визуальную корректность libass (рисует ли субтитры,
  обновляется ли правка) проверяет фаундер глазами — автоматом не верифицируется.

## 8. Approach B — Revideo (ЗАФИКСИРОВАНО НА БУДУЩЕЕ, НЕ делаем сейчас)
Полный OSS-WYSIWYG, превью=экспорт **включая кроп**, пиксель-в-пиксель.
- **Revideo** (MIT, github.com/redotvideo/revideo, ~3.8k★, на базе Motion Canvas): клип = TypeScript-
  композиция; `<Player>` рисует живьём, headless-рендер жжёт MP4 из ТОГО ЖЕ кода.
- Что даёт сверх Approach A: точный live-**кроп/reframe** в превью (не только субтитры), единый код
  превью+экспорт.
- Цена: **заменяет ffmpeg-рендер** (reframe-кроп/concat переписываем в Revideo-композицию; инвариант
  кадровой сетки переносим в композицию), Node-рендер-инфра, экосистема моложе ffmpeg. ~неделя+.
- **Когда делать:** после демо, когда «точный live-кроп» станет критичным. Отдельный спек+план.
- Почему НЕ Remotion: source-available, платно при >3 сотрудников — против требования «постоянно
  опен-сорс». Revideo (MIT) — корректная замена.

## 9. DoD (Approach A)
- `GET .../ass` отдаёт ASS текущего edit-state; unit зелёный.
- В модалке: видео = источник на моменте (двигаешь блок → едет); субтитры рисует libass из ASS;
  правка текста/стиля/позиции/пресета → libass обновляется живьём.
- ffmpeg-экспорт и reframe-инвариант не тронуты; `just check` зелёный.
- Approach B задокументирован (эта секция).
