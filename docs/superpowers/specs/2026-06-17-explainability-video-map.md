# Объяснимость + Карта видео + умная нарезка — Дизайн

> Статус: на ревью (2026-06-17). Это spec, НЕ план реализации. Релиз — всё одним заходом.
> Источник правды по реальности — `docs/README.md`. Перед кодом reframe/render — обязательно

## 0. Позиционирование (зачем) — наш дифференциатор

Конкуренты (OpusClip и т.п.) = «чёрный ящик»: загрузил → получил клипы + «virality score», без
объяснения и без понимания, что вообще в видео. **Quip отличается тем, что ПОНИМАЕТ всё видео и
ОБЪЯСНЯЕТ креатору, что происходит и почему момент стоит клипа — и помогает быстро найти/обрезать
свои моменты, а не только принять наши.**

Три столпа:
1. **Карта видео** — связный разбор всего видео (о чём, кто, ключевые моменты, тайм-коды), с
   привязкой к нашим клипам.
2. **Объяснимость на уровне момента** — в редакторе видно, ЧТО и ПОЧЕМУ интересно в каждом отрезке,
   с возможностью в один клик сделать оттуда клип.
3. **Умный ассистент** — агент понимает контекст всего видео и надёжно правит интервал/хук.

> **СКВОЗНОЕ ТРЕБОВАНИЕ (фаундер):** каждый UI-элемент этой фичи ОБЯЗАТЕЛЬНО проверяется ВИЗУАЛЬНО
> (десктоп И телефон) — ничего сломанного, всё адаптивно. Это часть «готово» в плане (см. §7).

## 1. Поток креатора (сценарии — думаем как креатор)

1. **Загрузка** → пайплайн делает клипы + **Карту видео**.
2. **Страница результатов:** сверху — **Карта видео** (читаемый разбор + кликабельные тайм-коды и
   ссылки на клипы), ниже — грид клипов (как сейчас, с «почему этот момент»).
3. Креатор читает карту → за 20 секунд понимает, о чём видео, и видит интересные моменты —
   **в т.ч. те, что мы НЕ вынесли в клипы**.
4. Открывает клип → редактор. Снизу — **обогащённая строка**: главы/темы всего видео, раскрываемые,
   с подсветкой интересных моментов и **клик→обрезка**.
5. Не нравится выбранный момент → в строке находит лучший → клик «Сделать клип отсюда» / «Подвинуть
   сюда» → интервал ставится (снап к словам, ≥20с).
6. **Агент:** «не нравится хук — глянь о чём видео и переделай в POV; обрезка резкая (начинает со
   слова "кстати") — поправь; сделай клип подлиннее». Агент это выполняет надёжно.
7. Субтитры / стиль / хук / кадр (пошотовый fit — уже есть) → правит быстро.
8. **Рендер → скачивает.**

## 2. Компонент A — «Карта видео» (страница результатов)

### Что это
Gemini-разбор всего видео в структуре «нарратив + главы + моменты + привязка к клипам». Пример
нарратива (как просил фаундер):

> «В начале (0:00–3:20) ведущий представляет гостя и тему. Затем (3:20–12:00) обсуждают X, и здесь
> — острый момент на **14:30**, где гость резко меняет позицию. Позже он рассказывает Y — из этого
> мы сделали **Клип №2**. Финал (25:00–27:00) — эмоциональная развязка.»

### UI (адаптив, мобила-first)
```
СТРАНИЦА РЕЗУЛЬТАТОВ
┌───────────────────────────────────────────────┐
│ 🗺  О чём это видео                    [скрыть] │
│  Связный разбор-нарратив с тайм-кодами 14:30    │  ← тайм-код кликабелен (плеер/прыжок)
│  и ссылками на «Клип №2» (ведут к клипу).       │
│ ─────────────────────────────────────────────  │
│  Главы:                                         │
│  ▸ 0:00–3:20  Вступление        · клипов: 0     │  ← раскрывается
│  ▾ 3:20–12:00 Спор о X          · Клип №2        │
│      Кратко: о чём глава…                        │
│      🔴 14:30–15:10  «резко поменял позицию»     │  ← цветной момент, клик → «сделать клип»
│      🟡 9:05–9:40    «сильная цитата»            │
│  ▸ 12:00–25:00 …                                 │
└───────────────────────────────────────────────┘
[ грид клипов ниже — как сейчас ]
```
- На мобиле: карта сворачивается в аккордеон, тайм-коды — крупные tap-таргеты (≥40px), горизонтальные
  переполнения скроллятся, ничего не наезжает.
- Кнопки на моменте: **«Сделать клип отсюда»** (создаёт/открывает черновик-клип на этом интервале,
  снап к словам, ≥20с) и **«Открыть в плеере»** (прыжок).

### Бэкенд (ложится на существующее)
- Уже есть: `app/editor/chapters.py` (`generate_chapters`), `generate_chapters_job`, эндпоинт
  `/jobs/{id}/chapters`, модель `ChaptersData`. **Расширяем это**, а не пишем с нуля.
- Новый/расширенный Gemini-вызов (structured output) из транскрипта → строит карту. Делается ПОСЛЕ
  select (знаем клипы → можем привязать `clip_ids`). Стоимость ~как select (~$0.01–0.03/видео,
  пренебрежимо vs транскрипция — см. BENCHMARKS).
- Хранение: расширить `ChaptersData`/добавить `VideoMap` в `models.py` → **codegen `just types`**
  (контракт). Поля: `narrative: str`, `chapters: [{start,end,title,summary,clip_ids,moments:[{start,
  end,label,why,kind}]}]`. `kind` = тип момента (tension/quote/emotional/insight/funny) → цвет.
- Генерация — в фоне (как `generate_chapters_job`), со статусом `pending/done/failed` и ретраем
  (как уже сделано для глав). Падение Gemini → видимый статус, не молчим (правило №8).

## 3. Компонент B — обогащённая строка в редакторе

### Что меняем
Сейчас нижняя `TimelineV2` показывает всё видео + темы (главы) пассивно. Делаем её **активным
инструментом «найди и обрежь»**:
- Каждая глава раскрывается → краткая суть («что происходит») + список **интересных моментов**.
- Моменты подсвечены ЦВЕТОМ по `kind` и подписаны живым языком (как просил фаундер: «потом ОНИ
  НАЧАЛИ СПОРИТЬ» — выделено цветом). Текст генерит Gemini (тот же VideoMap).
- **Клик по моменту → действие:** «Сделать клип отсюда» (новый клип) ИЛИ «Подвинуть текущий клип
  сюда» — интервал снапится к границам слов, длина ≥20с (если момент короче — расширяем симметрично
  до 20с в пределах source).
- Это переиспользует данные VideoMap (§2) → один источник правды, без второго Gemini-прохода.

### UI (адаптив)
```
РЕДАКТОР · низ (под таймлайном-скраббером)
┌───────────────────────────────────────────────┐
│ Темы видео                                      │
│ ▾ 3:20–12:00  Спор о X                          │
│    суть главы кратко…                            │
│    🔴 14:30 «они начали спорить»  [Сделать клип]│  ← клик-обрезка
│ ▸ 12:00–25:00 …                                 │
└───────────────────────────────────────────────┘
```
- Мобила: строка коллапсируется, не конфликтует с уже существующими `FitTimeline` (пошотовый кадр)
  и скраббером. Визуально проверить, что три «полосы» (скраббер, FitTimeline, темы) не сливаются и
  читаются на узком экране.

### Граница с уже сделанным
- `FitTimeline` (пошотовый fit) и эта «строка тем» — РАЗНЫЕ вещи: FitTimeline = кадр (горизонт/кроп
  по шотам), строка тем = смысл/нарезка по главам. В дизайне развести визуально (заголовки/иконки),
  чтобы юзер не путал (текущая претензия «непонятно»).

## 4. Компонент C — умный агент (доработка надёжности + контекста)

### Уже сделано (этот сеанс)
История чата, `get_surrounding_transcript`, стабильный флоу (`mode=ANY` + `respond_to_user`).

### Добавляем
1. **Контекст всего видео:** агент получает компактную сводку VideoMap (или тул `get_video_map`)
   → может «переделать хук с учётом всего видео», понимать, где момент в нарративе.
2. **Надёжные границы (фикс «двинь на 10с» тупил):**
   - В промпте жёстко: при сдвиге/триме держи `end ≤ source_duration` и длину в `[20с, 60с]`; если
     не помещается — двигай НАСКОЛЬКО можешь и ЧЕСТНО скажи юзеру, что упёрся в конец/лимит.
   - В `clip_state` агент уже видит `source_seconds` и текущий интервал — усилить промптом + проверить
     клампинг `set_interval`/`nudge_interval` (тест: сдвиг у конца видео не ломает длину, не вылетает
     за source). Если был баг клампинга — починить (TDD на pure-`compute_nudge`/clamp).
3. **Мульти-правка одним запросом** (уже поддержано циклом тулов): «переделай хук в POV + поправь
   резкий старт ("кстати") + сделай длиннее» → агент: `get_surrounding_transcript` (найти слово
   "кстати" → новый start после него) → `set_interval` (длиннее, ≥20с) → `regenerate_hook(style=pov)`
   с контекстом видео → `request_render` → `respond_to_user`.
4. **Хук с контекстом видео:** `regenerate_hook` опц. получает сводку VideoMap (а не только клип).
5. Рамки прежние: субтитры/кадр агент НЕ трогает (вне scope) — честно сообщает.

## 5. Компонент D — минимальная длина клипа = 20с (везде)

- `config.clip_min_sec`: 15 → **20**. Единый источник.
- `set_interval`/`trim`/`extend` (ops): уже клампят к `min_sec` — подхватят 20 автоматически; добавить
  тест границы.
- Фронт-слайдер/таймлайн (`TimelineV2`, `FitTimeline` не про длину): минимум ручки = 20с, попытка
  сделать короче → не даём + подсказка.
- Select-промпт (Gemini, stage2): целиться в клипы ≥20с (не отдавать огрызки).
- Агент: уважает 20с (см. §4).
- Клик-обрезка из карты/строки (§2,§3): момент короче 20с → расширяем до 20с в пределах source.

## 7. План-последовательность (для параллельных саб-агентов) + ВИЗУАЛЬНАЯ ПРОВЕРКА

Релиз цельный, но режется на НЕЗАВИСИМЫЕ домены:
1. **Бэкенд VideoMap** (Gemini + модель + джоб + эндпоинт + триггер) — основа для A и B.
2. **Мин. длина 20с** (config + select-промпт + ops-тест + слайдер) — изолированно.
3. **Агент** (контекст видео + надёжные границы + промпт + тесты) — поверх готового VideoMap-тула.
4. **Фронт: Карта видео** (страница результатов) — после контракта VideoMap.
5. **Фронт: обогащённая строка + клик-обрезка** (редактор) — после контракта VideoMap.

**Каждый фронт-таск ОБЯЗАН закончиться визуальной проверкой (десктоп + мобила):** скриншоты/прогон
через браузер, проверка что ничего не сломано, не наезжает, адаптивно (узкий экран), tap-таргеты
крупные. Это в DoD каждого фронт-таска. Гейт `just check` зелёный + codegen после `models.py`.

## 8. Решения (зафиксированы фаундером 2026-06-18) + риски

**Решения:**
1. **«Сделать клип отсюда»** из карты/строки: поддерживаем ОБА — по умолчанию кнопка **«Подвинуть
   текущий клип сюда»**, плюс отдельная **«Сделать новый клип»** (добавляет клип в список).
2. **Карта на мобиле — свёрнута по умолчанию** (аккордеон), на десктопе — раскрыта.
3. **Язык карты = язык видео** (как транскрипт).

**Риски:**
- **Стоимость/латентность VideoMap:** ещё один Gemini-вызов на видео (~$0.01–0.03). Приемлемо;
  генерим в фоне, не блокируем клипы.
- **Качество нарратива/моментов:** зависит от промпта; обязателен спот-чек на реальных видео (RU/EN)
  на боевых тестах перед мержем.
- **Контракт `models.py`:** изменение → `just types`; типы только codegen (правило).
