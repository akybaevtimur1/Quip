# Editor v3 — финальный редактор: страница, AI-карта, стили субтитров, split-screen

> Статус: брейншторм с фаундером 2026-06-12 (ответы получены, работа ночью автономно).
> Ветка: `feat/editor-v3`. База: Editor v2 (`2026-06-11-editor-v2-design.md`) +
> libass WYSIWYG (`2026-06-12-wysiwyg-libass-preview-design.md`).
> ⛔ Инвариант reframe НЕ трогаем: `docs/REFRAME_FPS_GRID_INVARIANT.md`.

## 1. Решения фаундера (зафиксированы дословно)

1. **Отдельная страница редактора** вместо модалки. КРИТИЧНО: возврат на главную должен
   быть безупречным — «не дай бог юзеру будет сложно возвращаться на основную страницу».
2. **AI-карта всего видео**: главы с описаниями (Gemini) + маркеры интересных моментов,
   «чтобы сразу понять контекст видео и придумать, в каком моменте потенциально можно
   сделать верт видео».
3. **Split-screen**: «нужно чтобы можно было и вручную в редакторе и чтобы агент сам делал»
   + обязательно поискать опен-сорс.
4. **Субтитры — всё обязательно**: драг позиции, цвет/размер/шрифт, 10+ пресетов, анимации
   слов. «Правка субтитров должна быть обязательно именно на видео».
5. **Плеер**: «подумай, точно ли не стоит менять плеер» — проработано, см. §3.
6. **Качество**: «ты буквально делаешь финальный вид — всё сразу профессионально и до готового».

## 3. Плеер превью (решение)

Остаёмся на `<video src=source.mp4>` + `LibassLayer` (SubtitlesOctopus). Причины:
- libass-канвас обязан сидеть поверх video-элемента и синхронизироваться вручную — внешняя
  обвязка (vidstack и пр.) это не решает, а fullscreen-перехват ломает (нужен fullscreen
  на КОНТЕЙНЕРЕ, чтобы канвас остался виден — грабля уже пройдена с CC overlay).
- Video.js v10 — бета; на «финальный вид» не строим.
- Нужны редакторские контролы, которых нет в плеерах: пауза-правка, переход по репликам.

**Строим `PreviewPlayer.tsx`** (свой контрол-бар, professional):
play/pause (klick по видео тоже), текущее время / длительность клипа, прогресс-скраб
ВНУТРИ клипа, fullscreen (на контейнере), луп клипа, состояние loading (большой источник).

## 4. Архитектура страницы

### Роут `/edit/[jobId]/[clipId]` (App Router)

```
┌──────────────────────────────────────────────────────────────┐
│ ← Все клипы │ Клип 2/5  ‹ ›       [Скачать] [Рендер] статус  │ ← хедер
├───────────────┬──────────────────────────────────────────────┤
│               │ Табы: Субтитры · Стиль · Кадр                │
│   PreviewPlayer│  Субтитры: список реплик, правка, вырез слов │
│   9:16 +      │  Стиль: пресеты 10+ (мини-превью), цвет/     │
│   libass +    │         размер/шрифт/позиция, анимация слов  │
│   контролы    │  Кадр: режим региона fill/fit/split (ручной) │
│               │                                              │
├───────────────┴──────────────────────────────────────────────┤
│ TimelineV2: главы AI (полоса с названиями) + 💡маркеры        │
│ моментов + волна + оранжевый блок шортса (drag/resize) + зум │
└──────────────────────────────────────────────────────────────┘
```

### Возврат без потери состояния (требование №1)
- Хедер: `<Link href={`/?job=${jobId}`}>← Все клипы</Link>` — deep-link уже работает
  (`f3d2249`): главная по `?job=` сама загружает задачу и показывает грид.
- Браузерный Back: переход на страницу редактора делается `router.push` с УЖЕ
  установленным `/?job=` на главной → Back возвращает на `/?job=<id>` → состояние
  восстанавливается из воркера (SQLite), ничего не теряется.
- Прямое открытие `/edit/<job>/<clip>` (закладка/F5) обязано работать: страница сама
  фетчит всё по URL-параметрам, никакой зависимости от стейта главной.
- Модалка `ClipEditorModal` удаляется; «Редактировать» на карточке → `router.push`.

## 5. AI-карта видео (главы)

### Контракт (models.py → just types)
```python
class Chapter(BaseModel):
    start: float; end: float
    title: str        # короткое название момента
    summary: str      # 1-2 предложения, что происходит
class ChaptersData(BaseModel):
    status: Literal["pending", "done", "failed"]
    chapters: list[Chapter] = []
    error: str | None = None
```

### Бэкенд
- `app/pipeline/stage_chapters.py` (или editor/chapters.py): Gemini structured output по
  транскрипту (та же обвязка retries/fallback-модели, что в `stage2_select`), промпт в
  `prompts/describe_chapters.v1.txt`. PURE-постобработка: сортировка, кламп в [0,duration],
  склейка перекрытий, минимальная длина главы.
- Кэш: `data/<job>/chapters.json` — генерится ОДИН раз (~$0.01-0.03), потом бесплатно.
- `GET /jobs/{job}/chapters`: есть кэш → 200 done; нет → старт фоновой генерации, 200
  pending; фронт поллит. Ошибка → status failed + error (правило №8, без тихих фолбэков).
- ⚠️ Free-tier квота Gemini (20/день) — ошибки квоты показываем честно, главы опциональны
  для UI (таймлайн работает и без них).

### Фронт (TimelineV2)
- Полоса глав над дорожкой: блоки с title, hover → summary + транскрипт.
- 💡-маркеры AI-сегментов с reason (есть в TimelineData) — клик = прыжок интервала.
- Зум/скролл для длинных видео (часовой подкаст), волна из плотности слов (есть).

## 6. Субтитры: стили, позиция, анимации

### Контракт
`CaptionStyle` уже имеет font/size/color/outline/box/alignment/margin_v/uppercase (v2).
Добавляем:
```python
class HighlightStyle(...):  # есть color/scale — добавить:
    animation: Literal["none", "karaoke_fill", "pop", "bounce"] = "karaoke_fill"
```
- Позиция = `margin_v` (+ `alignment`) — УЖЕ есть; драг на видео меняет их.
- Кастомизация поверх пресета: PATCH /edit с изменённым `captions.style` (механизм есть).

### Найденный баг: «Worker error: {}»
`subtitles-octopus.js:1043` — событие `error` Web Worker'а → libass дизейблится → ТИХИЙ
фолбэк на CSS-оверлей (правка субтитров «выглядит по-уебански» возможно ИМЕННО поэтому —
фаундер видел фолбэк, не libass). Ассеты public/ = node_modules (хэши совпали) → причина
рантаймовая (MIME wasm под Turbopack / шрифт / конкретный ASS). Диагностируем живьём
браузер-тулзами; починка libass-превью — ПРИОРИТЕТ фронт-работ.

### 10+ пресетов (`preset_seeds.py`, данные)
A Караоке-бокс · B Цветное слово · C Hormozi · D Чистая строка (есть) +
E MrBeast (жёлтый+чёрный жирный контур, pop) · F Neon (голубое свечение) ·
G Минимал (тонкий, без анимации, низ) · H Подкаст (бокс-плашка, верхняя треть) ·
I Karaoke Green (зелёная заливка слова) · J Bold White (огромный белый, bounce) ·
K Outline Pop (контурный, активное слово увеличивается) · L Lower Third (мелкий, прижат вниз).
Каждый = данные CaptionStyle+HighlightStyle; мини-превью в галерее рисуем CSS-ом по тем же
значениям (паттерн v2).

### Анимации слов → ASS (captions_v2)
- `karaoke_fill`: `\k` (есть).
- `pop`: на активное слово `\t(0,120,\fscx115\fscy115)` от 100.
- `bounce`: `\t(0,80,\fscy120)\t(80,160,\fscy100)`.
- Рендерится одинаково в превью (libass.wasm) и экспорте (ffmpeg libass). PURE-функция
  `word_animation_tags(style) -> str` + тесты.

### Он-видео правка (требование «не по-уебански»)
- Клик по активной реплике на видео → inline-textarea ПРЯМО на месте реплики (хит-зона
  уже есть в v2) → PATCH text_override → refetch ASS.
- Драг блока субтитров по вертикали → live-обновление margin_v (превью двигается сразу,
  на отпускание — PATCH + refetch ASS). Кламп в safe-area.
- Мини-тулбар при паузе: ✎ текст · ↕ позиция · 🎨 стиль.

## 7. Split-screen (два спикера верх/низ)

### Модель
`TrackRegion.mode` += `"split"`; для split добавляется вторая траектория:
`points_b: tuple[TrackPoint, ...] = ()` (второй спикер). Кадр: верх 1080×960 (кроп вокруг
cx_a) + низ 1080×960 (кроп вокруг cx_b), оба full-bleed, vstack.
Ручной override: `CropOverride.mode` += `"split"`, `+ center_b: float | None` (второй
центр; None → 0.75 если center=0.25 и т.п. — выбираем явные дефолты).

### Авто (plan_regions)
Сейчас: 2+ разнесённых лица → fit (блюр-рамки). Новое правило: РОВНО 2 устойчивых трека
лиц в шоте, оба разнесены → `split` (cx_a = левое/верхнее лицо… порядок стабилен по cx);
3+ лиц или нестабильные треки → fit как раньше. Fit ОСТАЁТСЯ для пейзажей/толпы —
требование фаундера «оставить горизонтальный вид для других ситуаций».
Кноб `REFRAME_SPLIT_ENABLED=true` + `REFRAME_SPLIT_MAX_FACES=2`.

### Рендер (stage5)
`build_smooth_filter`: ветка mode=="split" → два crop-чейна из одного trim-сегмента
(split=2), каждый `crop(w=src_h*(9/16)/2…)`→`scale=1080:960`, `vstack`, `setsar=1` →
в общий concat. Соблюдаем кадровую сетку (trim по кадрам) — инвариант не трогаем.
PURE-билдер + тесты (как build_smooth_filter-тесты).

### Ручной режим в редакторе (таб «Кадр»)
- Список регионов клипа (из reframe-данных editor'а) с текущим режимом; перекрытие
  пользователем: fill/fit/split → `CropOverride` (механизм есть в ClipEdit) →
  перерендер использует override.
- В превью кроп остаётся CSS-приближением (Approach A); точный вид — после рендера.

## 9. Ошибки (правило №8)
- chapters failed → бейдж «карта недоступна» + причина, таймлайн работает без глав.
- ASS refetch упал → тост + старый ASS остаётся.
- 409 → авто-reload edit-state (паттерн есть).
- split при <2 треков в регионе → 422 с пояснением (ручной override невозможен).
