# ClipFlow Editor v2 — дизайн (субтитры + таймлайн-выбор момента)

> Статус: УТВЕРЖДЁН фаундером 2026-06-11 (брейншторм + визуальные мокапы).
> Цель: довести редактор шортса до демо-уровня. Три части, общие контракты, потом
> параллельная сборка. НЕ трогаем инвариант reframe (`docs/REFRAME_FPS_GRID_INVARIANT.md`).

## 1. Цель и контекст

Длинное видео → ИИ нарезает шортсы → пользователь **редактирует в браузере**:
1. **Субтитры** выглядят чисто и интерактивно (караоке), выбор стиля-пресета, правка прямо на видео.
2. **Таймлайн**: шортс — подсвеченный блок на линии всего видео; двигать/растягивать; ИИ
   подсказывает сильные моменты и «что где происходит».
3. **Trim/extend/crop/перерендер** — уже есть (Editor Core), переиспользуем.

Принцип архитектуры: **один источник правды стиля → два рендера** (браузер CSS для превью/
правок; ASS-прожиг только на экспорте). Главный путь пайплайна УЖЕ отдаёт чистые клипы без
прожжённых субтитров — превью рисует субтитры живьём поверх.

## 3. Что СТРОИМ — три части

### Часть A — Caption Engine v2 (живой рендер + пресеты + inline-правка)

**A1. Рендер по модели стиля.** `CaptionOverlay` рисует субтитры из `CaptionStyle`+`HighlightStyle`
(не хардкод). **Убрать тёмную плашку** (корень «серого фона»); вместо неё контур+тень. Плашка —
только если пресет её явно задаёт (`box_color`). Safe-area: текст в нижних ~78% высоты, ширина
≤92%, авто-перенос, не уезжает за экран.

**A2. 4 пресета** (значения `CaptionStyle`/`HighlightStyle`), дефолт **A**:
- **A · Караоке-бокс**: белый+контур; активное слово в коралловой плашке (`#ff5a3d`).
- **B · Цветное слово**: белый+жирный контур; активное слово жёлтым (`#ffd23d`), без плашки.
- **C · Hormozi**: крупный размер, 1–2 слова, толстый контур; активное зелёным (`#34e36b`), scale>1.
- **D · Чистая строка**: фраза целиком, мягкая тень, `highlight=None` (без караоке).

Сид-пресеты доступны и фронту (TS-константы), и бэку (для экспорта). Один набор значений →
и CSS-превью, и ASS — внешний вид совпадает.

**A3. Inline-правка на видео** (выбранная модель):
- Клик по субтитру → правка текста на месте → `text_override` через `PATCH /edit`.
- Мини-тулбар у субтитра: **✎ текст** / **↕ позиция** (меняет `style.margin_v`, кламп в safe-area) /
  **🎨 стиль** (полоска пресетов A–D, `apply-preset`).
- Полоска пресетов под видео, активный подсвечен.

### Часть B — Таймлайн-редактор (выбор/сдвиг момента)

**B1. Данные таймлайна.** Новый `GET /jobs/{job_id}/timeline` → `TimelineData`:
```
TimelineData {
  duration: float                  # длительность источника (сек)
  segments: list[TimelineSegment]  # ВСЕ кандидаты ИИ (не только этот клип)
  words: list[Word]                # пословный транскрипт (для hover-подсказок)
}
TimelineSegment { clip_id: str|None; start: float; end: float; type: ClipType; score: float; reason: str }
```
Собирается из существующих `segments.json` + `transcript.json`. Дорогих ИИ-вызовов НЕТ.

**B2. Компонент `TimelineEditor.tsx`:**
- Дорожка = всё видео; «волна» (декоративно из плотности слов/громкости — MVP: из плотности слов).
- Маркеры сегментов ИИ по типу (цвет: hook/quote/peak/thought). Клик = прыжок плейхеда.
- **Блок шортса** = текущий `source_interval` клипа: drag = двигать, грипы краёв = resize. Кламп в [0,duration], мин. длина = `CLIP_MIN_SEC`, макс = `CLIP_MAX_SEC`.
- Плейхед + **hover-тултип**: транскрипт момента (из `words`) + ближайшая `reason` ИИ.
- Превью 9:16 и субтитры обновляются вживую при сдвиге (перемотка `<video>` + перерасчёт слов в окне).

**B3. Сдвиг/resize → бэкенд.** Новый op `set_interval`: `POST …/edit/set-interval {version, source_start, source_end}`
заменяет ПЕРВИЧНЫЙ интервал клипа (внешние границы). Trim-дырки внутри: сохраняются, если попадают
в новый диапазон, иначе отсекаются. Optimistic-lock (409) как у прочих op.

### Часть C — Экспорт берёт выбранный стиль

При «Скачать»/перерендере `render_clip_edit_job` компилирует ASS из текущего `CaptionTrack`
(стиль выбранного пресета) через `captions_v2` → прожиг. Уже почти есть; убедиться, что выбранный
пресет долетает до экспорта, и что главный грид-клип остаётся чистым (превью), а скачивание = прожиг.

**Эндпоинты (используем как есть):** `GET …/edit`, `PATCH …/edit`, `apply-preset`, `…/analysis`,
`…/render`, `GET/POST /presets`.

## 6. Обработка ошибок (правило №8)
- Бэкенд: нет segments/transcript → 404 (как `analysis`). Невалидные границы `set-interval`
  (start≥end, вне [0,duration], длина вне [min,max]) → 422/JobError, статус не портим.
- Фронт: 409 (version mismatch) → авто-reload edit-state (паттерн уже есть). Сетевой сбой → тост, не молча.
- Drag/resize клампится на клиенте ДО запроса (мгновенный отклик), бэкенд валидирует повторно.

## 7. Тестирование
- PURE-first (правило №3): `set_interval` (кламп/мин-макс/сохранение дырок) — unit ДО кода.
  Сборка `TimelineData` — unit на фикстуре segments+transcript. Маппинг стиля→CSS — unit на чистой ф-ии.
- Бэкенд-эндпоинты: TestClient (как `test_editor_api.py`), фон-задачи замокать.
- Фронт: `next build` + `just check` зелёные. Визуальная проверка (фаундер) на реальном/мок-клипе.
- НЕ юнит-тестим визуальное качество субтитров/таймлайна — глазами.
