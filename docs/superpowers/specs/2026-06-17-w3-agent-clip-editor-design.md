## 1. Цель и НЕ-цели

**Цель.** В редакторе ОТДЕЛЬНОГО клипа дать поле-чат, где юзер просит агента поправить КЛИП
естественным языком: «возьми момент чуть дальше», «обрезка не та, начни с вопроса», «хук не
цепляет — сделай шок». Агент **видит контекст клипа** (интервал, транскрипт клипа, текущий хук),
**работает тулзами** (двигает/расширяет интервал, перегенерирует хук, перезапускает рендер),
**показывает что думает и что делает**, и доводит клип до нужного состояния. Все правки —
недеструктивные (версионируются, откатываются).

**НЕ-цели (v1, явно вне скоупа).**
- ❌ Агент НЕ меняет субтитры (текст/стиль/тайминги/караоке) — ЖЁСТКАЯ граница (см. §8).
- ❌ Агент НЕ трогает reframe/кадрирование (инвариант кадровой сетки; нет reframe-тулзов в v1).
- ❌ Это НЕ чат-бот общего назначения: только про ЭТОТ клип; офф-топик → вежливый отказ.
- ❌ Не мультимодальный (видео в Gemini НЕ шлём — дорого; транскрипт+тайминги+хук достаточно).
- ❌ Не правит несколько клипов разом (скоуп = один открытый клип).

## 2. Пользовательский воркфлоу (что делает юзер / что видит)

1. Юзер открывает редактор клипа (`/edit/[jobId]/[clipId]`). Появляется вкладка/панель **«Agent»**
   (рядом с Captions / Style / Frame / Hook) ИЛИ выезжающая боковая панель чата.
2. Юзер пишет запрос: «начни на фразе про деньги и заканчивай чисто», жмёт Enter.
3. Поле ввода блокируется, появляется **активити-лента**: агент пишет короткие мысли
   («Смотрю транскрипт… фраза про деньги на 12.4с») и **карточки действий** («✂️ Сдвинул начало
   12.4с → 9.0с», «✨ Перегенерил хук: „Почему все теряют деньги“», «🎬 Запустил рендер»).
4. Превью клипа обновляется по мере правок (тот же edit-state, что и ручные правки → libass/превью
   реагируют мгновенно; рендер — в фоне со спиннером, как сейчас).
5. Агент заканчивает короткой сводкой: «Сдвинул начало на вопрос, обновил хук в стиле curiosity,
   перезапустил рендер. Глянь превью — ок?». Поле ввода разблокируется; юзер может уточнить.
6. В любой момент — кнопка **Stop** (как у джоба): останавливает агента, уже применённые правки
   остаются (недеструктивны, откатываются кнопкой Undo/историей версий). Списаний нет.
7. История чата сохраняется на клип: вернулся в редактор позже — переписка и действия на месте.

**Ключевые ощущения:** прозрачность (видно КАЖДОЕ действие и почему), обратимость (любое действие —
версионированная правка), контроль (Stop в любой момент), фокус (агент говорит только про клип).

## 3. UX-дизайн (как выглядит)

Панель чата — правый сайдбар редактора (на десктопе) / нижний лист (на мобайле, паттерн как у
существующих табов). Превью клипа остаётся видимым (агент правит — превью реагирует).

```
┌───────────────────────────── EDITOR ─────────────────────────────┐
│  [◀ clips]   Clip 2 / 6        [Undo] [Redo]      [Download ▾]     │
├───────────────────────────────────────────┬───────────────────────┤
│                                            │  AGENT                 │
│            ┌───────────────┐               │ ┌───────────────────┐ │
│            │               │               │ │ You:              │ │
│            │   PREVIEW      │               │ │ начни с вопроса,  │ │
│            │   (9:16)       │               │ │ хук послабее      │ │
│            │   + libass     │               │ └───────────────────┘ │
│            │               │               │   ⋯ thinking:          │
│            └───────────────┘               │   «ищу вопрос в начале»│
│   ───────────── timeline ─────────────     │ ┌───────────────────┐ │
│   [====clip====]                           │ │ ✂️ start 12.4→9.0s │ │
│                                            │ │ ✨ hook: "Почему…" │ │
│                                            │ │ 🎬 re-rendering…   │ │
│                                            │ └───────────────────┘ │
│                                            │ Agent: готово, глянь   │
│                                            │ ┌───────────────────┐ │
│                                            │ │ type a request…  ▶│ │
│                                            │ └───────────────────┘ │
│                                            │           [■ Stop]     │
└────────────────────────────────────────────┴───────────────────────┘
```

**Элементы:**
- **Лента сообщений**: чередование `user` (справа/акцент), `agent` текст (слева), `thinking`
  (приглушённый курсив, короткие строки активности), `action`-карточки (иконка + что сделано +
  было→стало, кликабельно → подсветить на таймлайне).
- **Состояния**: `idle` (поле активно) → `running` (поле disabled + спиннер + Stop) → `done`
  (поле активно, сводка) → `failed` (красная карточка + причина + «Retry») → `cancelled`
  (нейтральная карточка «Остановлено»).
- **Пустое состояние**: подсказки-чипы примеров («Сдвинь начало», «Сделай хук цепляющим»,
  «Покороче»). Сразу учит, ЧТО агент умеет (и неявно — границы).
- **Граница субтитров видна в UX**: если юзер просит про субтитры — агент карточкой объясняет «я не
  меняю субтитры, это вкладка Style» + (опц.) кнопка-переход на Style-таб.
- **Мобайл**: чат — нижний лист (как табы), превью `sticky` сверху (паттерн уже есть в редакторе).

## 4. Что умеет агент (тулзы)

Тулзы — тонкие обёртки над уже существующими pure-операциями (`editor/ops.py`) и хук-регеном (W4).
Агент НЕ дёргает БД напрямую; каждый тул применяется к АКТУАЛЬНОМУ edit-state (свежая версия) и
сохраняется через `store.save_edit` (optimistic-lock). Набор намеренно МИНИМАЛЬНЫЙ (боли фаундера):

| Тул | Назначение | Реализация |
|-----|-----------|------------|
| `get_clip_state()` | Прочитать текущий интервал, длину, транскрипт клипа, текущий хук, лимиты | read edit + `clip_words` + meta |
| `set_interval(start_sec, end_sec)` | Поставить окно клипа (двигать/обрезать) | `ops.set_interval` (кламп min/max/дюрация) |
| `nudge_interval(edge, delta_sec)` | Сдвинуть край на ±delta («чуть дальше/раньше») | `ops.apply_extend` / пересчёт + set_interval |
| `regenerate_hook(style_hint?)` | Перегенерить хук под текущий интервал (стиль W2) | `regenerate_hook` (W4) |
| `set_hook_text(text)` | Поставить точный текст хука (если юзер продиктовал) | патч `hook.text` |
| `request_render()` | Запустить пере-рендер из edit-state | `render_clip_edit_job` (фон) |
| `respond(message)` | Финальный/уточняющий текст юзеру (НЕ действие) | завершить шаг |

**Намеренно НЕ даём (v1):** изменение субтитров, reframe/crop, пресеты, аспект, trim-дырки в
середине, удаление клипа. (Можно добавить позже — отдельные тулзы, отдельный скоуп.)

Каждый тул возвращает агенту **структурированный результат** (новое состояние / что изменилось /
был ли кламп / ошибка). Кламп или ошибка валидации = НЕ молчим: результат честно говорит агенту
«начало уклампилось до 0.0, клип теперь 60с (макс)», агент сообщает это юзеру.

## 5. Архитектура

### 5.1 Поток (high-level)
```
Frontend (chat)                Worker web (FastAPI)             Modal agent_edit_job (spawned)
  │ POST /agent/start  ───────▶ создать agent_run (running),
  │   {message}                 dispatch.spawn("agent_edit_job",  ──▶ агент-луп:
  │                              run_id) → function_call_id           Gemini(tools) ⇄ apply tool
  │ ◀── {run_id}                сохранить fc_id в agent_run            → пишет события/статус в
  │                                                                     agent_run (инкрементально)
  │ GET /agent/{run_id}  ──────▶ вернуть статус + сообщения/действия ◀──┘ (фронт поллит ~1с)
  │   (poll)
  │ POST /agent/{run_id}/cancel ▶ FunctionCall.cancel() → InputCancellation
```

### 5.2 Агент-луп (бэкенд, `app/agent/clip_agent.py`)
Manual function-calling Gemini-луп (НЕ structured output):
1. Собрать `system_prompt` (роль + границы + стиль) + контекст клипа (интервал/длина/транскрипт/хук)
   + историю сообщений этого run.
2. Вызвать Gemini с `tools=[function_declarations]`.
3. Ответ:
   - содержит `function_call` → выполнить тул (apply к edit-state, save), записать `thinking`
     (короткая преамбула из текста ответа, если есть) + `action`-событие в agent_run, добавить
     `function_response` в историю, **goto 2**;
   - чисто текст → это `respond` → записать `agent`-сообщение, **завершить run (done)**.
4. **Hard cap**: ≤ `AGENT_MAX_STEPS` (дефолт 8) тул-вызовов на один запрос → защита от петель;
   при достижении — финализируем с честным сообщением «остановился на лимите шагов».

### 5.3 Состояние / персистентность
Новая таблица `agent_runs` (Supabase + локальный SQLite-зеркало), идемпотентно по `run_id`:
- `run_id` (uuid), `job_id`, `clip_id`, `user_id`, `status` (running|done|failed|cancelled),
  `function_call_id` (для отмены), `messages` (jsonb: упорядоченный лог событий — user/thinking/
  action/agent/error), `error`, `created_at`, `updated_at`, `cancellable` (bool).
- История чата клипа = последние run'ы по (job_id, clip_id) → лента восстанавливается при открытии.
- Запись событий — инкрементальная (агент пишет по мере шагов → фронт-поллинг видит прогресс live).

### 5.4 Переиспользование (НЕ изобретаем фон/отмену)
- **Спавн/фон** — `dispatch.spawn("agent_edit_job", run_id)` (как `render_job`); Modal-функция
  `agent_edit_job` в `deploy/modal/worker.py` (mirror render_job: cpu=2, timeout=600, secrets).
- **Отмена** — `function_call_id` + `modal.FunctionCall.from_id(id).cancel(terminate_containers=False)`
  → `InputCancellation` (BaseException) → луп выходит ДО любой платной операции (её и нет) → `$0`.
- **Поллинг** — фронт-хук как `useJob`/render-poll (интервал ~1с пока `running`).

### 5.5 Биллинг (инвариант: агент = $0 минут)
- Агент-луп = Gemini micro-вызовы (как W4-реген, ~$0.0003/шаг) + БЕСПЛАТНЫЙ editor-рендер
  (`render_clip_edit_job` не вызывает `_meter`). **Минуты НЕ списываются НИКОГДА** в агент-пути.
- Отмена в любой момент = $0 (нет `_meter` в пути вообще). Совпадает с денежным инвариантом сессии.
- (Себестоимость Gemini-вызовов агента — копейки; не метрим, как и прочие правки редактора.)

## 7. Сценарии и крайние случаи (предугадываем максимум)

| # | Сценарий | Поведение |
|---|----------|-----------|
| 1 | Офф-топик («какая погода») | Вежливый отказ одним сообщением, без тулзов, run=done |
| 2 | Просьба про субтитры («сделай субтитры крупнее») | Объясняет: субтитры не меняю → вкладка Style; без тулзов |
| 3 | Просьба про кадр/кроп | Аналогично: reframe вне моих возможностей (v1) |
| 4 | `set_interval` за границы источника | `ops` клампит в [0,duration]; результат честно говорит о клампе; агент сообщает юзеру |
| 5 | Интервал < min или > max сек | `clamp_interval` правит; агент сообщает «клип уклампился до 60с» |
| 6 | Параллельная ручная правка во время run (версия ушла) | Тул читает СВЕЖУЮ версию перед каждым применением; при `EditConflict` на save → перечитать раз и повторить; если опять конфликт → честно остановиться с сообщением |
| 7 | Юзер шлёт новый запрос пока агент работает | Поле ввода disabled во время `running`; новый запрос — после завершения/Stop. (Одно run на клип.) |
| 8 | Gemini транзиентно недоступен (429/5xx) | Ретраи (как в `call_gemini_structured`); устойчиво → run=failed с причиной, правки до сбоя остаются (откатываемы) |
| 9 | Петля тулзов (агент дёргает бесконечно) | Hard cap `AGENT_MAX_STEPS=8` → финализация с сообщением |
| 10 | `request_render` упал | Карточка действия = failed с причиной; edit-state уже применён (рендер отдельно); юзер видит ошибку, не списано |
| 11 | Stop в середине | `InputCancellation` → run=cancelled; уже применённые правки ОСТАЮТСЯ (недеструктивны, Undo доступен); $0 |
| 12 | Юзер закрыл вкладку в середине | Run живёт в Modal (как джоб); вернулся — поллинг покажет итог/историю. Закрытие НЕ отменяет |
| 13 | Пустой интервал (нет слов) — нечего хукать | `regenerate_hook` вернёт ошибку «нет слов»; агент сообщает, предлагает расширить интервал |
| 14 | Галлюцинация аргументов (start>end, нечисло) | `ops` рейзит `JobError` → результат тула = ошибка → агент видит и исправляется (или сообщает) |
| 15 | Агенту нужна доп. инфо | Может `respond(вопрос)` и завершить run, ожидая следующего сообщения юзера |
| 16 | Юзер просит «верни как было» | v1: направляем на Undo (история версий). (Тул `undo` — кандидат v2.) |
| 17 | Рендер уже идёт, агент зовёт ещё | Guard: один рендер на клип (reuse render-status); агент ждёт/сообщает |
| 18 | Двойной POST /start (дабл-клик) | Идемпотентность: если для клипа уже есть `running`-run → вернуть его, не плодить |
| 19 | Очень длинный/спамный запрос | Лимит длины сообщения (напр. 2000 симв); тримминг |
| 20 | Биллинг: агент не должен списать минуты | Путь не содержит `_meter` вообще; тест-гард |

## 8. Предостережения / инварианты (ОБЯЗАТЕЛЬНО соблюдать)

1. **Субтитры неприкосновенны.** Агент НЕ имеет тулзов на субтитры. Система-промпт явно запрещает.
   Тест: набор тулзов не содержит ничего про captions/replies/style/highlight.
2. **Reframe-инвариант цел.** Нет reframe-тулзов; агент-правки идут через тот же `set_interval`/
   рендер-путь, что и ручные (кадровая сетка не трогается). `REFRAME_FPS_GRID_INVARIANT.md` в силе.
3. **Биллинг $0.** Ни одной `_meter`/`record_usage`/`deduct_payg` в агент-пути. Отмена = $0.
4. **Недеструктивность.** Каждое действие = версионированная правка edit-state (Undo/Redo работают).
   Cancel НЕ авто-откатывает (правки реальны и обратимы вручную) — это явно в UX.
5. **preview == export.** Агент меняет только интервал/хук → существующий рендер-путь (тот же ASS,
   тот же libass-превью). Никакого второго пайплайна.
6. **EditConflict уважается** (optimistic-lock); параллельные правки не теряются молча (сц. №6).
7. **Без тихих фолбэков** (правило №8): ошибка тула/Gemini/рендера → видимое событие/статус, не молча.
8. **Анти-чатбот**: система-промпт жёстко скоупит на редактирование ЭТОГО клипа; офф-топик/субтитры
   → вежливый отказ, без тулзов.
9. **Идемпотентность run** по `run_id`; один активный run на клип.

## 9. План реализации (фазы)

**Фаза 0 — контракт.** `models.py`: `AgentRunStatus`, `AgentEvent`, `AgentRun`, `AgentStartBody`.
`just types`.

**Фаза 1 — бэкенд (TDD на чистой логике).**
- `app/agent/tools.py` — декларации тулзов (Gemini function declarations) + диспетчер
  `apply_tool(name, args, job_id, clip_id) -> ToolResult` (обёртки над `ops`/`regenerate_hook`/
  рендер). PURE-валидация аргументов тестируется.
- `app/agent/clip_agent.py` — агент-луп (Gemini function-calling, step-cap, сбор контекста,
  запись событий). Сетевой Gemini — тонкая обёртка; парсинг/контроль шага — pure-тесты.
- `app/agent/store.py` (или в `editor/store.py`) — CRUD `agent_runs` (local SQLite + cloud).
- `db.py`/миграция `0007_agent_runs.sql` (Supabase MCP) — таблица `agent_runs`.
- `app/tasks.py` — `agent_edit_job(run_id)` (склейка: луп + статус + cancellable + НИ ОДНОГО `_meter`).
- `app/main.py` — `POST /jobs/{id}/clips/{cid}/agent/start`, `GET …/agent/{run_id}`,
  `POST …/agent/{run_id}/cancel`. (mirror cancel-эндпоинта: ownership, 409/idемпотентность.)
- `deploy/modal/worker.py` — `agent_edit_job` Modal-функция.

**Фаза 2 — типы.** `just types`.

**Фаза 3 — фронт.** `apps/web/lib/api.ts` (startAgent/getAgentRun/cancelAgent),
`components/editor/AgentTab.tsx` (лента + ввод + Stop + состояния), хук-поллинг `useAgentRun`,
интеграция в `ClipEditorScreen` (новый таб/панель; превью реагирует на edit-state).

**Фаза 4 — тесты + реальный прогон.** Unit (диспетчер тулзов, step-cap, парсинг, границы),
API (мок Gemini: start→poll→done, cancel→cancelled, офф-топик, субтитры-отказ),
**реальный Gemini-прогон** агент-лупа на кэш-клипе (двинуть интервал + регенерить хук),
`just check` зелёный, прогон на реальном видео из загрузок.

**Фаза 5 — деплой + верификация.** `modal deploy`; проверить эндпоинты живые; (миграция через MCP).

## 10. План тестирования

- **Pure/unit**: `apply_tool` (каждый тул: happy + кламп + ошибка), step-cap, сбор контекста клипа,
  парсинг function_call, анти-петля, валидация аргументов.
- **API (мок Gemini)**: старт→поллинг→done; cancel→cancelled (+ $0 — нет meter); идемпотентность
  двойного старта; офф-топик→отказ-без-тулзов; субтитры→отказ; EditConflict→graceful.
- **Биллинг-гард**: агент-путь не вызывает `_meter`/`record_usage`/`deduct_payg` (тест).
- **Реальный Gemini**: на кэшированном транскрипте/клипе — «сдвинь начало на 3с раньше и обнови хук»
  → проверить, что интервал реально сдвинулся, хук перегенерился, события записались.
- **Реальное видео**: прогнать на последнем видео из загрузок (как в W1) — агент правит клип,
  рендер обновляется, глазами проверить качество/стабильность.
- **`just check` зелёный** перед каждым коммитом.
