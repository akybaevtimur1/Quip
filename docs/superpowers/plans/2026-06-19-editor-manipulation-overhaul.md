# Editor manipulation overhaul — hook/subtitle drag, resize, font stability, preset scroll

> Хэндофф-спека для СВЕЖЕГО сеанса. Задача провалилась ~5 раз, потому что интерактивные баги
> чинились ВСЛЕПУЮ (редактор под авторизацией, мок их не показывает). **Главное правило: сначала
> получить доступ к ЖИВОМУ редактору и воспроизвести баг, потом чинить, потом проверить скрином.**
> Не угадывать. Тяжёлые куски — Opus-4.8 саб-агентами с прод-промптами (тесты + проверка).

## Что просил фаундер (дословный смысл)
1. **Драг неудобный / блоки хвата слишком большие / не тактильно.** Хук и субтитры тянутся плохо.
2. **Нельзя менять ШИРИНУ блока хука/субтитров сбоку** (сузить блок, не меняя размер текста — текст
   перенесётся на больше строк). Сейчас угловая ручка меняет только размер ШРИФТА.
3. **Нельзя двигать по X и в угол** (свободное позиционирование). Сейчас только вертикаль.
4. **То же для субтитров.**
5. **Шрифт «отскакивает»:** меняешь шрифт — применился — резко вернулся к прежнему, надо менять ещё раз.
6. **Лента стилей (пресетов) не листается** с ноута (колесо мыши не скроллит горизонтально).

## Архитектура (факты — соблюдать)
- Субтитры/хук рендерятся **libass** (ASS) И в превью редактора (subtitles-octopus / libass-wasm),
  И в ffmpeg-экспорте воркера — **один движок**, поэтому валидный ASS работает в обоих. Инвариант
  WYSIWYG: превью == экспорт. Экспорт ВСЕГДА из Python-ASS (`services/worker/app/editor/captions_v2.py`).
- **Горизонтали в модели НЕТ:** текст центрируется (ASS \an2 субтитры снизу / \an8 хук сверху),
  управляется только `margin_v` (вертикаль) + `size`. Свободный X/Y и ширина блока (#2,#3) ТРЕБУЮТ
  расширения модели + ASS-рендера → **кросс-стек** (см. ниже).
- Рамка манипуляции (`apps/web/components/editor/OverlaySelectionBox.tsx`) уже строится по РЕАЛЬНОМУ
  bbox от libass (воркер libass-wasm постит `{x,y,w,h}` на кадр через `onSubRects` из `LibassLayer.tsx`).
  Драг: тело = двигать (вертикаль, commit `margin_v`), угол = ресайз шрифта (commit `size`),
  grab-offset уже учтён. Pure-логика тайминга/координат — под тестами.
- ASS-стиль строится в `apps/web/lib/assStyle.ts` (мгновенный локальный патч Style-строк) — ЗЕРКАЛО
  `captions_v2.py`. Недавно добавлены: `SINGLE_WEIGHT_FONTS`/`assBoldFlag` (Bold=0) и (в воркере)
  `resolve_font_for_text` (кириллический фолбэк). 6 новых шрифтов уже забандлены и работают.

## Корни проблем (локализовано в этом сеансе — проверить repro перед фиксом)
- **#5 «отскок шрифта»:** гонка в `apps/web/components/editor/ClipEditorScreen.tsx`. `editCaptions`
  (≈L376) делает оптимистичный `patchAssStyles` (мгновенный новый шрифт) + дебаунс-`scheduleFlush`
  (300мс). `flushCaptions` (≈L335) после PATCH зовёт `refreshAss()` (≈L177) → `getClipAss` →
  `setAssText(серверный ASS)`. Гипотеза: серверный ASS приезжает со СТАРЫМ шрифтом (PATCH не успел/не
  сохранил font, ИЛИ reconcile перетирает свежий локальный патч). **Воспроизвести в живом редакторе,
  посмотреть сетевые ответы getClipAss/patchClipEdit, найти точную причину (systematic-debugging).**
- **#6 скролл ленты:** компонент стиля/пресетов (галерея из `GET /presets`). Горизонтальная лента
  `overflow-x-auto` не скроллится колесом мыши на ноуте → добавить `onWheel`: `deltaY → scrollLeft`
  (+`e.preventDefault`). Найти строку галереи в `StyleTab.tsx`/`HookTab.tsx`/`ClipEditorScreen.tsx`.
  Низкий риск, можно вслепую, но проверить.
- **#1 тактильность:** рамка садится на bbox текста (он широкий) → хват «большой». Возможно нужно:
  меньшая/чётче ручка, hit-zone не на всю ширину, мгновенный 1:1 (setPointerCapture уже есть).
  Проверять руками в живом редакторе.
- **#2/#3 ширина блока + X/Y (КРОСС-СТЕК, самое большое):**
  - Модель (`services/worker/app/models.py` → codegen `just types`): добавить хуку/субтитрам
    горизонтальную позицию (напр. `margin_x`/центр-доля) + ширину-блока (wrap-width, напр.
    `margin_l/r` или доля). Без изменения существующих дефолтов (обратная совместимость).
  - Воркер ASS (`captions_v2.py` + зеркало `assStyle.ts`): применить X через `\pos`/выравнивание +
    ширину через `margin_l/r` (libass переносит внутри). НЕ ломать кадровую сетку/инвариант.
  - Фронт (`OverlaySelectionBox.tsx` + `ClipEditorScreen.tsx`): ручки — углы (move/scale), БОКОВЫЕ
    (ширина блока), тело (move X+Y). Commit в новые поля. Драг императивный (без ререндера на move),
    commit на pointerup (как уже сделано).
  - ВЕРИФИЦИРОВАТЬ: ширина в превью (libass) == ширина в экспорте (ffmpeg). Это и есть смысл WYSIWYG.

## Definition of done
Хук и субтитры: хватаются мгновенно и тактильно; двигаются по X и Y (в т.ч. в угол); боковая ручка
меняет ШИРИНУ блока (текст перетекает) без изменения размера шрифта; угловая — размер; смена шрифта
СТАБИЛЬНА (не отскакивает); лента пресетов листается колесом. Превью == экспорт. `just check` зелёный,
проверено в ЖИВОМ редакторе скринами.
