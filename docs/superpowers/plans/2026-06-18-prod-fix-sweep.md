## 1. Убрать плашку «о чём видео» (TopicStrip) из РЕДАКТОРА 🟢
- **Проблема:** плашка тем под видео в редакторе — лишний шум, мешает. Нужна ТОЛЬКО на странице
  результатов (главной), пока что.
- **Корень (расследование):** в редакторе рендерится `apps/web/components/editor/TopicStrip.tsx`
  (через `ClipEditorScreen.tsx`). На странице результатов — отдельный компонент
  `apps/web/components/VideoMap.tsx`. Это ДВА разных компонента → можно убрать один, не трогая другой.
- **Делаем:** убрать рендер `TopicStrip` из редактора (`ClipEditorScreen.tsx`). `VideoMap` на
  результатах НЕ трогаем.
- **✅ Готово:** в редакторе под видео НЕТ плашки тем; на странице результатов карта видео осталась
  и работает; `tsc`/eslint зелёные.

## 2. Переосмыслить «thinking»/«action» теги в чате агента 🟢
- **Проблема:** внутренние теги «thinking»/«action» выглядят по-ишному и бесполезно; рядовой юзер их
  не замечает/не понимает. Нужно объяснять иначе или сделать понятнее.
- **Корень:** `apps/web/components/editor/AgentTab.tsx` рендерит роли `user/agent/thinking/action/
  error` как теги (события из `runs_store`, бэкенд `app/agent/loop.py`).
- **Делаем:** оформить «мысли»/«действия» как НЕНАЗОЙЛИВЫЙ человекопонятный прогресс (свернуть в
  компактный «шаги/работаю…» с раскрытием по клику; человеческий текст вместо `thought`/`action`).
  Бэкенд событий не ломаем — меняем подачу на фронте.
- **✅ Готово:** обычный юзер видит понятный прогресс без жаргона; детали доступны по желанию, но не
  шумят; чат читается чисто.

## 3. Чат агента: сообщение юзера показывать МГНОВЕННО 🟢
- **Проблема:** отправленное сообщение не появляется сразу → юзер шлёт повторно → дубли/недопонимание.
- **Корень:** лента, вероятно, обновляется только по polling (`getAgentRun`/`getActiveAgentRun`),
  без оптимистичного локального эха (см. `AgentTab.tsx`, `lib/api.ts`).
- **Делаем:** оптимистично добавлять сообщение юзера в ленту СРАЗУ при отправке; на время отправки
  дизейблить инпут/кнопку (анти-дубль); при ошибке — откат/пометка.
- **✅ Готово:** отправил — мгновенно видно в чате; повторная отправка во время обработки невозможна.

## 4. ВОТЕРМАРК для free — серверно, без обхода (БИЗНЕС-КРИТИЧНО) 🟢 (нужен `modal deploy`)
- **Проблема:** надо гарантировать, что free-юзер НИКАК не получит клип без вотермарки.
- **Корень (КРИТИЧНО):** `PlanLimits.watermark` есть в `services/worker/app/billing.py` (free=True),
  но прожиг **НИГДЕ не реализован** — в рендер-пути (`stage5_render`/`run.py`) вотермарки нет. То есть
  СЕЙЧАС free-юзеры получают ЧИСТЫЕ клипы. `max_resolution` (720/1080) тоже не энфорсится.
- **Делаем:** в рендер-пути читать план юзера СЕРВЕРНО (по `jobs.user_id` → `profiles.plan` →
  `resolve_plan`) и для free **прожигать вотермарку в пиксели КАЖДОГО клипа** (ffmpeg) + кап 720p.
  Решение принимается на сервере — клиент не может отключить. Прокинуть `watermark`/`max_resolution`
  в per-clip фан-аут (`reframe_render_clip`). ⚠️ ПЕРЕД правкой stage5/reframe — читать
  `docs/REFRAME_FPS_GRID_INVARIANT.md`, не ломать кадровую сетку. Pure-логику (нужна ли вотермарка,
  кламп разрешения) — под TDD.
- **Проверить bypass-векторы:** нет ли API/preview/source-пути отдать готовый клип без вотермарки
  (клипы — продукт; preview/source — это исходник, не клип).
- **⏸️ Решение фаундера:** дизайн вотермарки (текст «Made with Quip» / лого, угол, прозрачность, размер).
- **✅ Готово:** free-аккаунт → каждый клип с прожжённой вотермаркой и ≤720p; платный → без вотермарки,
  1080p; снять через UI/API невозможно; `just check` зелёный (вкл. тест pure-логики).

## 5. Привязка видео к АККАУНТУ (не к устройству) 🟢 (по схеме — ок, верифицировать)
- **Проблема:** убедиться, что видео привязаны к аккаунту юзера, а не к устройству.
- **Корень (расследование):** `migrations/0001`: `jobs.user_id uuid references auth.users(id)`,
  RLS `(select auth.uid()) = user_id`. Видео **уже привязаны к аккаунту**, не к устройству.
- **Делаем:** верифицировать, что воркер ставит `user_id` из токена (`app/auth.py`) на создании джобы;
  убедиться, что нет device/localStorage-идентичности. Задокументировать как инвариант.
- **✅ Готово:** подтверждено с доказательством (file:line); вход с другого устройства → те же видео.

## 6. Анти-абьюз мультиаккаунтов (free = 2 видео) 🟢 (выбор: verified email + disposable; нужен Supabase «Confirm email» = ON + `modal deploy`)
- **Проблема:** «2 free видео» легко абьюзить сменой аккаунтов → нужен барьер.
- **Корень:** `handle_new_user`-триггер сидит free-профиль на КАЖДУЮ запись `auth.users`; ничто не
  привязывает грант к верифицированной личности (нет обяз. подтверждения email/телефона/устройства/IP).
- **⏸️ Решение фаундера (выбрать стратегию — см. вопрос):** напр. обязательный verified email перед
  выдачей free-кредитов; free только для Google OAuth (email/пароль — без free); device/browser
  fingerprint; лимит по IP на регистрации/free-джобы; карта для free; капча. Баланс сила↔трение↔конверсия.
- **✅ Готово:** выбранный барьер внедрён и проверен; повтор «новый аккаунт = ещё 2 видео» затруднён
  выбранным способом; честные юзеры не страдают.

## 7. Драг/ресайз хука и субтитров + артефакты + кривые контролы 🟢
- **Проблема:** хук/субтитры в превью тянутся лагово, не «хватаются» напрямую, самопроизвольно
  меняется масштаб; артефакты в полноэкранном режиме; в блоке «плашка»/стиль тон и цвет стоят не ровно,
  много кривого.
- **Корень:** оверлеи/драг в `apps/web/components/editor/PreviewPlayer.tsx` (+ `LibassLayer.tsx`);
  контролы стиля — `apps/web/components/editor/StyleControls.tsx`.
- **Делаем (frontend-агент):** прямой `pointer-capture` драг (следует за курсором мгновенно), разделить
  move/resize (скейл не прыгает при перемещении), нормализация координат не зависит от ререндера/размера
  элемента; починить трекинг оверлея в фуллскрине; выровнять контролы стиля (flex/grid, единые обёртки).
- **✅ Готово:** хватается мгновенно, тянется плавно, масштаб не прыгает, фуллскрин без артефактов,
  контролы ровные; проверено визуально (десктоп+мобила).
