# Оценка качества нарезки (gate Phase 0 → Phase 1)

Цель: понять, **достаточно ли хороши клипы**, прежде чем строить инфру Phase 1.
Метрика: **Q = (usable + fixable) / total ≥ 60%** (план §7.4).

## Как оценивать (анти-bias: смотри клип ДО того, как глянул `reason`/score)

1. Сгенерить таблицу: `cd services/worker; uv run python -m app.eval <job_id>`
   → `data/<job_id>/eval_sheet.csv` (строка на клип, колонки C1–C8 пустые).
2. Открыть CSV (Excel/Google Sheets). Для КАЖДОГО клипа посмотреть видео **целиком, со звуком,
   на телефоне вертикально**, проставить C1–C8 = 0/1 (сомнение = 0) и `reason_agrees` = 0/1.
3. Посчитать: `uv run python -m app.eval <job_id> --score` → бакеты + **Q** + GO/NO-GO.

## Критерии C1–C8 (бинарно, «1» когда…)

| | Критерий | «1» когда |
|---|---|---|
| C1 | Standalone | понятен без контекста исходника |
| C2 | Hook ≤2с | в первые 2с вопрос/заявление/конфликт/число |
| C3 | Clean start | старт на границе фразы, не с середины слова |
| C4 | Clean end | конец на завершённой мысли |
| C5 | Face 9:16 | лицо в кадре весь клип, не обрезано (или FIT-режим: контент виден) |
| C6 | Captions accurate | ≤1 ошибка субтитров на клип |
| C7 | Captions sync | рассинхрон < ~150мс |
| C8 | Length 15–60с | (sweet spot 20–45с) |

`reason_agrees` = согласен ли ты с объяснением модели (валидация моата объяснимости).

## Бакеты (killers!)

- **REJECT** — killer (C1=0, или C5=0 с грубо сломанным кадром, или C6=0) ИЛИ сумма ≤3.
- **FIXABLE** — провалены только тримабельные (C2/C3/C4/C8), сумма ≥5, без killers (чинится Phase 2 trim-редактором).
- **USABLE** — сумма ≥7, без killers.

## GO / NO-GO (план §7.4)

- **GO:** Q ≥ 60%, usable/total ≥ 25%, mean(reason_agrees) ≥ 70%, C6-killer < 15%.
- **NO-GO → диагностика винтика:** низкий Q при ОК reason_agrees и провалах C2/C3/C4 → итерация
  промпта (`prompts/select_moments.v1.txt`) + snap-to-sentence. Низкий и Q, и reason_agrees →
  переписать промпт / сменить модель. C6 плохой на RU/шуме → swap провайдера транскрипции.

## Что нужно от фаундера
Golden set — несколько **in-scope** роликов (один говорящий, есть речь):
- EN чистый подкаст/соло, EN экспертный влог, RU подкаст (проверка языка).
Кэш транскрипции (K2) сделает повторные прогоны бесплатными — пока без него каждый прогон платит.
