## R1 — Reframe 2.0 (АКТИВНАЯ БОЛЬ, делаем первым) ⬅
Проблема сейчас: «флеши» = смена кропа не попадает точно в склейку камеры; перебивки/b-roll
(без лица) кропаются узким слайсом (трава вместо «показать широко»). Корень: грубый ffmpeg
scene-порог (то ложные, то пропуски) + кроп одним «выражением во времени» (промах на кадр).
Как у Opus (подтверждено ресёрчем): точный детект склеек (~97%) + reframe ПО-ШОТНО + адаптивный масштаб.
- **Точные склейки:** PySceneDetect (есть в asd-зависимостях) вместо сырого ffmpeg-порога;
  позже TransNetV2 для сложных переходов.
- **Рендер ПО-ШОТНО (сегменты + concat):** кроп постоянен внутри шота, меняется ровно на
  границе = на склейке камеры → флеши уходят by design. Каждый шот независимо.
- **Адаптивный масштаб/режим на шот:** лицо есть → тайт-кроп на ГОВОРЯЩЕГО (ASD, уже есть);
  перебивка/нет субъекта → ШИРЕ или fit (весь кадр), не слайс. (Opus: масштаб «гуляет» по моментам.)
- DoD: на comedy01 + Comedians-in-Cars нет флешей, перебивки показаны широко; кадры до/после.

## R2 — Не только подкасты + мягкий empty-state
Сейчас stage2 (Gemini) заточен на «реплики одного спикера» → на не-речевых видео (влог,
геймплей, музыка, экшен) возвращает 0 → «интересных моментов не нашлось» (фаундер: бесит).
- Промпт выбора моментов расширить: визуальные хуки/экшен/смена темы, а не только речь.
- Empty-state мягче + диагностика «почему» (язык/мало речи/тип контента), не глухое «ничего нет».
- DoD: не-подкаст видео даёт клипы; «нашлось 0» только когда реально пусто.

## R3 — Стили субтитров (выбор)
Opus: несколько стилей прожжённых субтитров (жирные, karaoke-подсветка слова, мемные).
- N пресетов ASS-стиля (шрифт/контур/цвет/анимация подсветки) + выбор в UI на клип/прогон.
- Karaoke-подсветка текущего слова (ASS \k-теги) — фирменная фишка шортсов.
- DoD: 2–3 стиля на выбор, видно в UI, прожигаются корректно.

## R4 — Virality-score на клип

## R5 — Авто-хук (первые ~5с)
Opus добавляет хук-оверлей в начало клипа. Текст-хук из reason/первой реплики + стиль.

## R6 — Ручная правка кропа/масштаба в UI
Opus: можно подвинуть/масштабировать кадр вручную на клип. Пост-MVP.

## Порядок

## Технологии-ориентиры (ресёрч)
- Детект склеек: PySceneDetect (эвристика, кадроточно) + TransNetV2 (ML, добор пропусков).
- Active-speaker: LR-ASD (уже вендорено, `app/asd/`) + MediaPipe-детект (быстрый).
- Opus: ~97% scene detection (обученные модели, не один порог) — наш ориентир по точности.
