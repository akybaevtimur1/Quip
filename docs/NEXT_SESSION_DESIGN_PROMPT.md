# Промпт для следующей сессии (вставь это первым сообщением)

Привет. Сейчас твоя задача — **полностью переработать ДИЗАЙН (UI/UX)** продукта Quip (редактор
коротких клипов из подкастов). У нас местами проблемы и в UX, и в UI — выглядит не везде на уровне,
кое-где по-ИИшному и шаблонно. Ты должен сделать так, чтобы дизайн стал **максимально запоминающимся,
уместным и НЕ «ИИшным»** — как у живого премиум-продукта, а не сгенерированный шаблон.

## С чего начать (строго по порядку)

1. **СНАЧАЛА запроси у меня логин в браузере.** Тебе нужен залогиненный Playwright (плагин), иначе ты
   не увидишь реальные авторизованные страницы (дашборд, редактор). Открой `http://localhost:3000`,
   попроси меня войти вручную (Google/email) в окне Playwright, дождись «готово», и только потом работай.
   Локальный dev-сервер фронта (`pnpm --filter web dev`) обычно крутится на **:3000** (он же ходит в
   ПРОД-воркер — реальные данные). Если не запущен — подними сам.
2. **Прочитай `docs/README.md` ПЕРВЫМ** (реальность + карта деплоя + что не ломать), затем `CLAUDE.md`
   (правила), затем `docs/JOURNAL.md` (верхние записи 2026-06-22/23 — что уже сделано). В `design-md/`
   лежат вендоренные дизайн-референсы — посмотри, как делают другие, но НЕ копируй слепо.
3. **Сверяйся с кодом перед правкой** — не угадывай. Открывай нужные файлы по индексу из README.

## Задача (объём)

## Как работать (ОБЯЗАТЕЛЬНЫЕ правила — те же, что всегда)

- **Результат — главное.** Не экономь токены и время — у меня практически бесконечные лимиты. Работай
  на результат: **не успокаивайся, пока всё реально не работает и не выглядит хорошо** (глазами через
  Playwright-скриншоты, а не «на словах»).
- **Используй ДОХУЯ саб-агентов (форков) на Opus 4.8** — столько, сколько нужно, хоть десятки. Ты —
  ОРКЕСТРАТОР: режь работу на НЕЗАВИСИМЫЕ домены (разные страницы/компоненты), запускай ПАРАЛЛЕЛЬНО,
  затем интегрируй и верифицируй. Ключевой урок прошлой сессии: **разделяй форки по ГЕЙТАМ и ФАЙЛАМ** —
  два web-форка дерутся за общий `tsc` (видят незаконч. файлы друг друга); web-форк + worker-форк
  уживаются (разные гейты). Центральные общие файлы (`ClipEditorScreen.tsx`, `dashboard/page.tsx`) —
  по одному писателю за раз. Коммиты сериализуй (pre-commit `just check` глобален).
- **Делай на совесть** — это живой прод, но в ЭТОЙ сессии **деплоить НЕЛЬЗЯ** (см. ниже). Всё равно
  пиши код так, будто идёт в прод: без тихих фолбэков, типы только codegen, гейт зелёный.
- **`just check` зелёный перед КАЖДЫМ коммитом** (ruff+mypy+tsc+eslint+тесты+anti-drift). Коммить
  ТОЛЬКО из PowerShell с PATH refresh (pre-commit хук зовёт `just`). **Частые локальные коммиты** —
  чтобы можно было откатиться.
- **Обширные проверки глазами**: после каждого заметного изменения — скриншоты через Playwright на
  localhost, на тупые И неочевидные косяки (особенно адаптив/состояния). Не сходится 2× → СТОП, разберись
  по корню (скилл `systematic-debugging`), не угадывай.
- **Без скоупа сверх задачи.** Не тащи лишние рефакторы. Не трогай палитру.
- **Пиши мне по-русски.**
