## 4. Юнит-экономика и МАРЖА

**Полный стоимостный прогон sample01 (~33 мин) на текущем стеке:**

| Стадия | Стоимость | Доля |
|---|---|---|
| 0 Import | $0 | — |
| 1 Transcribe (Deepgram) | ~$0.14 | ~90% |
| 2 Select (Gemini flash) | ~$0.016 | ~10% |
| 3 Reframe / 4 Captions / 5 Render | $0 (local CPU) | — |
| **ИТОГО** | **~$0.16 / видео (33 мин)** | |

- Экстраполяция на **60 мин**: Deepgram ~$0.26 + Gemini ~$0.03 = **~$0.29/видео**.
- **Таргеты плана:** cost/video ≤ **$2** (ceiling $3). Сейчас **~$0.16–0.29** — глубоко в пределах.
- **Маржа:** при цене $5–15/видео и cost ~$0.16–0.29 → **gross margin ~94–99%** (без инфры).
- **Доминанта стоимости = транскрипция** ($0.14 из $0.16). Главный рычаг маржи —
  self-host WhisperX (→ $0) и кэш транскрипции по hash(source) при итерациях промпта.
- Инфра (не в cost/video): Railway (worker) + Vercel (web) — фикс/мес, добавить при деплое.

## 6. Спайк: active-speaker reframe (LR-ASD) — замеры на CPU (2026-06-08)

Цель: наводиться на ГОВОРЯЩЕГО, а не на крупнейшее лицо. Прогнал готовый репозиторий
[LR-ASD](https://github.com/Junhua-Liao/LR-ASD) (MIT) как есть на **этой машине (CPU, без GPU)**.
Вход — 15с 16:9 мультиспикер-клип из comedy01.

| Метрика | Значение |
|---|---|
| Параметры: ASD-модель (LR-ASD) | **0.84M** (дёшево) |
| Параметры: face-детектор (S3FD) | **22.46M** (~85MB весов) |
| Параметры ИТОГО | **23.3M** |
| Wall-time (весь пайплайн репо) | **169.9с на 15с клип (~11× реалтайма)** |
| Peak RAM (RSS, вкл. ffmpeg) | **823 MB** |
| Качество | ✅ корректно выделил говорящего (2 дорожки score>0, 13 — <0) |

**КЛЮЧЕВОЙ разбор времени (lean in-memory прогон, без файлового плумбинга):**

| Стадия | Время (15с клип, CPU) |
|---|---|
| извлечение кадров@25fps + аудио + загрузка | 4.6с |
| **S3FD детект лиц (375 кадров)** | **128.9с ← УЗКОЕ МЕСТО** |
| трекинг | 0.0с |
| кроп лиц (in-memory) + ASD-форвард | 12.3с |
| ИТОГО | 145.7с |

Узкое место — НЕ плумбинг, а сам S3FD-детектор лиц (тяжёлый VGG, ~3 fps на CPU).

**🔑 Решение — MediaPipe вместо S3FD.** У нас УЖЕ есть быстрый детектор (MediaPipe BlazeFace).
Замер на тех же 375 кадрах: **3.1с (121 fps, 277 лиц) — ~42× быстрее S3FD.**

| Подход | Детект лиц | Полный lean-прогон 15с клип (CPU) | Качество ASD |
|---|---|---|---|
| Репо как есть (S3FD + ffmpeg-плумбинг) | 128.9с | ~170с | эталон (score спикера +0.4) |
| Lean + S3FD | 128.9с | 145.7с | эталон |
| **Lean + MediaPipe (наш)** | **3.0с** | **15.3с** ✅ | score сжат (+0.1) — нужен тюнинг кропа |

**Вывод для маржи:** lean + MediaPipe + ASD (0.84M) = **15.3с/клип на CPU (~реалтайм)** —
полностью приемлемо. Разбивка: extract 5.3с + MediaPipe 3.0с + track 0с + crop+ASD 6.9с.

**Открытый вопрос — КАЧЕСТВО:** ASD-модель обучена на S3FD-кропах; на MediaPipe-кропах
(они теснее) разделение «говорит/молчит» СЛАБЕЕ (score спикера +0.1 vs +0.4). Рычаги:
(1) подстроить `cropScale` под MediaPipe-боксы (вернуть привычный модели кадр); (2) брать
ОТНОСИТЕЛЬНЫЙ argmax на план (нам нужен лишь «кто громче говорит», не калибровка);
(3) S3FD на ПОНИЖЕННОМ fps (~6fps) + интерполяция боксов → ~48с/клип, но эталонное качество.
Подтверждать — визуально на рендере при интеграции.

> Спайк-окружение (изолированный venv с torch CPU + S3FD) — в `services/worker/tmp/spike/`
> (gitignored). Все цифры — реальные прогоны на этой машине (CPU), не оценки.

## 5. Качество: как меряем (рубрика §7.1)

Поклипово 0/1 по C1–C8 (standalone, hook, clean start/end, face-framing, captions
accuracy/sync, длина), бакеты USABLE/FIXABLE/REJECT, метрика **Q = (usable+fixable)/total**.
GO-gate: Q ≥ 60%, usable ≥ 25%, reason_agrees ≥ 70%. Полную оценку (golden set 6 источников,
≥18 клипов) делаем на этапе H/J, результаты — отдельной секцией сюда.

### 7.2 Reframe-анализ (лица+склейки) — CPU vs GPU

| Инстанс | analyze (тот же 55с сэмпл) | $/клип |
|---------|---------------------------|--------|
| CPU × 8 ядер | 75с | **$0.0079** |
| A10 (спайк) | 51с | $0.0157 |

GPU быстрее лишь в 1.46× (за счёт более мощного хост-CPU инстанса), но **в 2× дороже**, а сам
reframe стоит копейки. **MediaPipe считает на CPU (XNNPACK) в обоих случаях** → видеокарта тут
не нужна. Reframe → CPU.

### 7.3 РЕШЕНИЕ по железу (источник правды)

- **На старте — НУЛЬ GPU.** Reframe = CPU (Modal), транскрипция = Deepgram (API), рендер = CPU
  ffmpeg. Весь джоб помещается в ОДНУ Modal CPU-функцию. Это самый дешёвый возможный вариант.
- **Видеокарта = рычаг «на потом»**: при объёме >~1000 видео/мес перенести транскрипцию на
  **self-host faster-whisper large-v3 на L4** (НЕ A10) → −$0.12/видео.
- **A10 не берём никогда** для нашей нагрузки — подтверждённая переплата.
- **Поток юзеров:** Modal scale-to-zero ($0 в простое) + авто-параллель контейнеров +
  `max_containers` (потолок против разгона счёта) + посекундный биллинг. Переплаты-в-простое нет.
