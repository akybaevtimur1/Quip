# Quip — Обзор проекта (всё в одном файле, для брейнсторма)

> Самодостаточный обзор: что делает продукт, какие сервисы, как работает пайплайн, лимиты и
> цифры, биллинг/маржа, хранилище, ключевые решения и ПОЧЕМУ. Источник правды по «текущей
> реальности» — `docs/README.md`; код — ссылки ниже. Цифры актуальны на 2026-06-16.

## 1. Что делает Quip
Берёт **одно длинное видео/подкаст** и нарезает из него несколько **вертикальных шортсов** (9:16),
каждый с **прожжёнными субтитрами**, **хуком** (топ-плашка), **confidence-score** и человекочитаемой
причиной «почему зайдёт». Есть редактор клипа (правка интервала, субтитров, хука, кропа, пресеты) и
скачивание (с субтитрами / без). Вход — загруженный файл (сейчас основной) или YouTube-ссылка (скрыта).

## 3. Как работает пайплайн (по шагам)
1. **Загрузка.** Браузер просит у воркера `POST /jobs/upload-url` → грузит файл **напрямую в R2**
   (presigned PUT; файлы >100 МБ — **multipart**, части параллельно) → `POST /jobs/{id}/upload-complete`
   спавнит обработку. (Не через воркер: один большой POST на Modal рвался.)
2. **Spawn.** Web-контейнер (scale-to-zero) спавнит долгоживущую Modal-функцию `upload_job`/`run_job`
   (`timeout=10800` = 3 ч, `cpu=4`, `memory=4096`) — иначе scale-to-zero убил бы фон.
3. **Stage 0 — Import.** Скачивает source из R2 → `source.mp4` (умный remux, без лишнего транскода) +
   `source.wav` (16 кГц моно, для транскрипции) + `ffprobe` (длина/fps/размер). Лимит длины **3 ч**.
4. **Гейт квоты** (после probe, ДО платного): если минут не хватает — джоб падает до оплаты Deepgram.
   **Граница FREE→PAID:** всё до сюда бесплатно для нас; транскрипция — первый платный шаг.
5. **Stage 1 — Transcribe.** `source.wav` → Deepgram → слова с таймингами (`transcript.json`). Кэш по
   хэшу аудио (повтор = $0). В Gemini уходит ТОЛЬКО транскрипт, **не видео**.
6. **Stage 2 — Select** (главный гейт качества). Gemini (structured output) по **индексам слов**
   (не таймкодам — нет «придуманного времени») выбирает моменты + пишет `hook` + `why_works` + `score`.
   Постобработка: снап к границам предложений, маппинг в секунды, фильтр длины 15-60с, анти-overlap,
   обрезка до N клипов.
7. **Stages 3-5 — per-clip** (на каждый клип): **reframe** (сцены→шоты→режим fill-кроп/fit-блюр-рамки/
   split, слежение за спикером через ASD@25fps, ресемпл в нативный fps) → **render** (ffmpeg: trim по
   кадрам + кроп + прожиг ASS-субтитров+хука) → заливка клипа в R2.
8. **Preview-прокси.** Весь source перекодируется в лёгкий `preview.mp4` (≤720p H.264) для быстрого
   редактора. Артефакты (meta/segments/transcript) → Postgres.
9. **Редактор.** Правка интервала/субтитров/хука/кропа/пресетов → ре-рендер клипа через `render_job`.
10. **Скачивание.** Клип на `cdn.quip.ink` (вечный публичный URL); `Content-Disposition: attachment`
    → реально сохраняется. Варианты: с субтитрами (captioned) / без (clean) / SRT.

## 4. Лимиты и цифры (точные)
- **Размер загрузки:** ≤ **10 ГБ** (`SourceForm.MAX_UPLOAD_MB`, просто guard; >100 МБ → multipart).
  Реальный предел — **длина 3 ч** (`billing.MAX_VIDEO_MINUTES = 180`).
- **Клипы:** до **30** на видео (режим **Auto** «сколько найдётся ≤30» или **Custom 1-30**). Длина
  одного клипа **15-60с** (`clip_min_sec=15`, `clip_max_sec=60`, sweet spot 20-45с).
- **Таймаут обработки:** **3 ч** на джоб (Modal `run_job`/`upload_job`), CPU=4 ядра, RAM=4 ГБ.
- **Окно presigned-загрузки:** **6 ч** (медленный аплинк на 10 ГБ).
- **Транскрипция (Deepgram):** аудио-only (3 ч ≈ 345 МБ WAV, под лимитом 2 ГБ); ~$0.26/час.
- **LLM (Gemini Flash):** вход = транскрипт. 1 ч видео ≈ 9000 слов ≈ **15-20k input-токенов**; выход
  ≤ `llm_max_output_tokens=16000`. **≈ $0.005-0.01 за видео.** (Точные токены логируются в `runs.jsonl`.)
  3 ч → ~45-60k input (Flash тянет 1M контекст). Видео в Gemini НЕ шлём — стоимость = размер транскрипта.
- **Стоимость обработки на видео (start-стек):** ≈ **$0.33** (Deepgram $0.26 + Gemini $0.03 + Modal compute).
- **Скорость:** для длинных видео доминирует preview-транскод + per-clip рендер (CPU); cpu=4 даёт ~линейный
  выигрыш. boto3 TransferConfig (64 МБ/20 потоков) ускоряет 5-10 ГБ download R2→worker.

## 5. Биллинг, прайсинг, маржа
- **Модель — кредиты.** 1 кредит = 60 минут исходника. Тарифы (источник правды `app/billing.py` ⇄
  `lib/plans.ts`): **Free** $0 / 2 кредита · **Starter** $10 / 10 · **Pro** $25 / 30 · **PAYG** $2 (разовый).
- **Платежи — Polar** (Merchant-of-Record, сам считает налоги), вебхук → `profiles.plan` / payg_credits.
- **Маржа (с учётом стоимости $0.33/видео):** Starter ~67%, Pro ~60%, PAYG ~84%. Хранилище R2 ест ~1 п.п.
- **Списание минут — только по факту готовых клипов** (`record_usage` после `set_done`). **Отмена джоба
  (Stop) во FREE-фазе = $0** (заряд недостижим). PAYG-кредиты не сгорают, идемпотентны по job_id.

## 6. Хранилище R2 (размеры и ретеншн)
- **На джоб ≈ 0.4-1.85 ГБ:** `source.mp4` (70-90% всего!) + `preview.mp4` (~10%) + клипы (~80 МБ).
  2-часовой 1080p подкаст ≈ **1.85 ГБ** → ~5-6 таких забивают бесплатные 10 ГБ.
- **R2-цена:** 10 ГБ free, дальше **$0.015/ГБ-мес**, **egress бесплатный** (ключевая причина выбора R2 —
  раздача клипов через CDN ничего не стоит). Операции — копейки.
- **Ретеншн (важно):** разовая оплата → вечное хранение = безлимитный рост. Поэтому **ежедневный Modal
  Cron `cleanup_stale_sources`** удаляет `source.mp4`/`preview.mp4` старше **60 дней** (клипы — вечны).
  Source нужен лишь редактору для ре-рендера; через 60 дней редактирование старого клипа недоступно
  (надо перезалить). Срезает рост хранилища ~80%.

## 7. Ключевые архитектурные решения и ПОЧЕМУ
- **Загрузка напрямую браузер→R2 (не через воркер):** один большой POST на Modal-web рвался (truncated
  multipart → 400 + CORS). R2 = Cloudflare edge, надёжно. **Multipart** >100 МБ — обход 5 ГБ single-PUT
  лимита + параллельно/resume.
- **Modal spawn (не BackgroundTask):** scale-to-zero web-контейнер погасил бы фон-задачу на полпути.
  Долгоживущая отдельная функция переживает закрытие вкладки → **джоб виден в recent, по возврату —
  прогресс/результат**.
- **Gemini по индексам слов, не таймкодам:** снимает класс ошибок «придуманное время».
- **ASD на 25fps + ресемпл в нативный fps; границы регионов в НАТИВНОЙ кадровой сетке:** ⛔ **инвариант**
  (`docs/REFRAME_FPS_GRID_INVARIANT.md`) — иначе «флеши» на видео ≠25fps (на 25fps баг невидим, тесты
  зелёные). Пан внутри fill-региона — piecewise-LINEAR (плавно), границы не трогаются.
- **Шрифты:** хук Unbounded не имеет bold-начертания → `Bold=0` (иначе libass на воркере подменял
  семейство на Montserrat). Один ASS для libass-превью и ffmpeg-прожига → **preview == export (WYSIWYG)**.
- **cpu=4/memory=4096 на воркере:** дефолтный ~1/8 ядра душил ffmpeg-транскод длинных видео.
- **Stop-кнопка только во FREE-фазе:** биллинг безопасен by-construction (заряд после `set_done`; Modal
  `InputCancellation` = BaseException минует except → set_done недостижим). Платную стадию не отменяем.

## 8. Что НЕ ломать (инварианты)
- ⛔ **Reframe кадровая сетка** (Δ=0) — `docs/REFRAME_FPS_GRID_INVARIANT.md`.
- **Контракт типов** — codegen из `services/worker/app/models.py` (`just types`); `packages/shared/*` руками не трогать.
- **Money paths** — `billing.py` ⇄ `lib/plans.ts` менять ВМЕСТЕ; Polar-вебхук подпись = сырые байты секрета.
- **Без тихих фолбэков** — ошибка → JobError/failed/явный лог.
- **WYSIWYG** — превью (libass.wasm) == прожиг (ffmpeg+libass): один и тот же ASS.

## 10. Открытые узлы (для брейнсторма)
- Резкий обрыв клипа (снап конца только +5 слов; нет хвостовой паузы для лупа) — `stage2_select.snap_end_index`.
- Лимит длины: моменты >max молча дропаются, модель не знает реальный max.
- Хуки описательные, не эмоционально-стилевые (POV/мем/инфо/шок).
- Хук не регенерится при сдвиге клипа (субтитры — да, транскрипт-driven; хук = сохранённый оверлей).
- Латентность скачивания: sync-рендер на web-контейнере (async-export спроектирован, не сделан).
- Лаги редактора (per-change раунд-трип + libass re-render) — кандидат на client-side ASS.
(Детали и план — `docs/NEXT_SESSION_BRIEF.md`.)
