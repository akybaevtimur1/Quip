# За что юзер ПЛАТИТ: анти-заложник, эффекты, экспорт (брейншторм 2026-06-13)

> Контекст: editor v3 готов и смержен. Фаундер посмотрел реальные шортсы подкастов
> (5 скриншотов, разбор ниже) и поставил вопрос: «как захватывать юзеров именно нашей
> платформой и заставить платить». Главный страх: юзер-заложник — заплатил, а доделать
> видео не может ни у нас, ни в CapCut.

## 1. Что показали примеры фаундера (паттерны рынка)

| # | Паттерн | Что на скрине | У нас сейчас |
|---|---------|---------------|--------------|
| 1 | Сочные пословные субтитры | Rogan-подкаст: слова разными цветами/градиентом, эмодзи 😱 в строке | Частично (12 пресетов, караоке, анимации; нет пер-слово цветов/эмодзи) |
| 2 | **B-roll врезки** | Спикер сверху + UFC-бой снизу под слово «WHIPPED»; фото Безоса фуллскрин под «podcasts» | НЕТ — главный разрыв |
| 3 | Карточный лейаут | Чёрный фон, заголовок-хук сверху, два окна спикеров, подпись снизу | Частично (split без рамок; нет «карточки» с заголовком) |
| 4 | Кастомные оверлеи | Стрелки, doodles, before/after коллаж, плавающие картинки | НЕТ (и не должны делать первыми — см. экспорт) |

Вывод: рынок = «субтитры + смысловые врезки + лейауты». Чистые субтитры-на-кропе —
коммодити (Submagic/OpusClip/Klap делают все).

## 2. Анти-заложник: юзер ВСЕГДА может доделать видео (стратегический принцип)

Иерархия экспортов (от дешёвого к дорогому в реализации):

1. **Чистый mp4 без прожига** — уже есть технически (рендер без ass_name). Дать кнопку
   «Скачать без субтитров». ~час работы.
2. **SRT/ASS файл субтитров** — у нас уже есть compile_ass; SRT-экспорт тривиален
   (слова+тайминги есть). CapCut/Premiere/Resolve импортируют SRT. ~час работы.
3. **🔑 Экспорт ПРОЕКТА CapCut** — OSS [pyCapCut](https://github.com/GuanYixuan/pyCapCut)
   (+ [pyJianYingDraft](https://github.com/GuanYixuan/pyJianYingDraft) для 剪映) генерит
   draft-папку CapCut из Python: видео-сегменты с таймингами, ТЕКСТОВЫЕ дорожки
   (=наши субтитры остаются редактируемым текстом!), эффекты, анимации. Юзер
   открывает CapCut → наш клип уже нарезан, кадрирован, с субтитрами — и ВСЁ можно
   двигать. Никто из конкурентов в нашем сегменте это не даёт (OpusClip отдаёт только
   XML для Adobe/Resolve — про-юзерам, не CapCut-аудитории).
   Риски: формат draft неофициальный (ломается при апдейтах CapCut — следить за
   репо), Windows-only автоэкспорт (нам не нужен — нужна генерация папки). Спайк ~день.
4. (позже, про-сегмент) OTIO/FCPXML для Premiere/Resolve.

Принцип продукта: **мы продаём СКОРОСТЬ до 90% результата + СВОБОДУ доделать 10% где
угодно**. Платный план без экспортов = ловушка = отток.

## 3. B-roll/врезки — следующий большой кусок ценности

У нас уникально хорошая позиция: уже есть **пословный транскрипт + AI-главы + Gemini-обвязка**.
Пайплайн авто-B-roll:
1. Gemini по транскрипту клипа: «выбери 2-4 момента, где врезка усилит удержание;
   на каждый — поисковый запрос + тип (фото/видео/эмодзи)» (structured output, как select).
2. Источники (бесплатные API, OSS-friendly): **Pexels API** (видео+фото, free),
   Pixabay, Giphy (стикеры/эмодзи-анимации), Wikimedia (личности — фото Безоса!).
3. Рендер: ffmpeg overlay поверх (полэкрана/фуллскрин с лёгким zoom-движением
   (zoompan), наш single-pass фильтр уже умеет слоить).
4. Редактор: таб «Врезки» — список предложенных (превью, момент на таймлайне),
   вкл/выкл, замена (поиск по своему запросу), своя картинка аплоадом.
МВП-скоуп: фото/гифки фуллскрин + полэкрана, 2-4 на клип, ручное курирование. Видео-сток — v2.

## 4. Лейауты-карточки (дёшево, заметно)

Расширить режимы кадра (у нас уже регионная модель!): `card` — фон-цвет/блюр,
заголовок-хук сверху (Gemini уже пишет reason/title!), окно(а) спикера с радиусом,
подпись снизу. Это один новый билдер в _region_chain + стили. Дифференциатор скрина №3.

## 5. Субтитры до уровня примеров

- Пер-слово цвет/эмодзи: Gemini размечает «ударные» слова (то же место, где главы) →
  ASS-теги цвета на слово + эмодзи (шрифт Noto Color Emoji в fontsdir + libass —
  проверить рендер эмодзи в libass: возможен моно-глиф; спайк).
- Пресеты-«бренды» юзера: сохранение своих пресетов УЖЕ есть (POST /presets) — поднять в UI.

## 6. Монетизация (гипотеза к проверке)

- **Free**: 2-3 клипа/видео, вотермарк, без экспортов проекта.
- **Pay ($15-25/мес как Submagic/Klap)**: безлимит клипов, БЕЗ вотермарка, SRT+чистый
  mp4, **CapCut-проект**, B-roll, свои пресеты/шрифты.
- Якорь ценности в онбординге: «от ссылки до 5 готовых шортсов за 3 минуты + проект
  CapCut в подарок». Платят за: экономию 2-4 часов на эпизод + свободу.
- Метрика правды: доля клипов, скачанных/экспортнутых (= реально использовали).

## 7. Порядок работ (предложение)

| Приоритет | Что | Почему | Оценка |
|---|---|---|---|
| 1 | Стабилизация багов (список в HANDOFF §0.1) | без этого платить не будут | 0.5-1 сессия |
| 2 | Экспорт-пакет: чистый mp4 + SRT/ASS кнопками | анти-заложник минимум, копейки | 0.5 сессии |
| 3 | Спайк pyCapCut: наш клип+субтитры → draft, открыть в CapCut руками | киллер-фича, проверить ДО обещаний | 1 сессия |
| 4 | B-roll MVP (фото/гифки, Pexels+Giphy, таб «Врезки») | разрыв с примерами рынка | 1-2 сессии |
| 5 | Лейаут «card» | дёшево, видно в ленте | 0.5 сессии |
| 6 | Auth+оплата+деплой (K1 очередь, инфра-дизайн уже в памяти) | собственно «заставить платить» | отдельный трек |

## 8. Решения фаундера (зафиксированы 2026-06-13)

1. **ЦА: ОБА сегмента сразу** (соло-подкастеры + SMM-агентства) → в роадмапе обе
   экспорт-ветки: CapCut-draft первым (массовее/дешевле), FCPXML/OTIO для про — вторым.
   Учесть расфокус-риск: маркетинг/онбординг всё же ведём от подкастера.
2. **B-roll: автоматом, легко выключить** — клипы сразу выходят «как у людей с примеров»
   (вау на первом прогоне), таб «Врезки» в редакторе = вкл/выкл/замена каждой.
3. **Следующая сессия: баги → экспорты → CapCut-спайк** (порядок §7 подтверждён).
   Открытым остался только вопрос вотермарка на free (решить при auth/оплате).
