## 📚 Документация ⇄ код (ОБЯЗАТЕЛЬНО — держим в синхроне)

1. **В начале сессии прочитай `docs/README.md`.** По нему открывай НУЖНЫЙ код, чтобы свериться и
   понять логику ПЕРЕД правкой — не угадывай (индекс маршрутизирует: «задача → что читать»).
2. **Меняешь поведение — обнови доку В ТОМ ЖЕ заходе.** Особенно ПРОДУКТОВЫЕ решения (цены, лимиты,
   флоу, планы, фичи): правь источник правды сразу (`billing.py` ⇄ `lib/plans.ts`; раздел «реальность»
   в `docs/README.md`).
3. **Не можешь обновить полностью — повесь баннер `⚠️ OUTDATED` в шапку дока** с указателем на источник
   правды. Доки НЕ должны молча расходиться с кодом.
4. **Заметное решение/фича → одна строка в `docs/JOURNAL.md`** (история/ADR). Изменил «текущую
   реальность» (деплой/биллинг/инфра/крупная фича) → обнови раздел в `docs/README.md`.
5. **Перед коммитом, меняющим поведение, спроси себя:** «не врёт ли теперь какой-то док?» → почини
   или пометь. Это часть определения «готово».

## 🤖 Рабочий стиль: саб-агенты (ОБЯЗАТЕЛЬНО для всего)

- **По умолчанию используй саб-агентную систему** для любой нетривиальной работы: ресёрч, аудит,
  исследование кодовой базы, параллельные правки по доменам, проверка/ревью. Ты — ОРКЕСТРАТОР, не делай
  всё последовательно сам.
- Основа — скилл **`superpowers:dispatching-parallel-agents`**: режь задачу на НЕЗАВИСИМЫЕ домены (чтобы
  агенты не правили одни и те же файлы), запускай ПАРАЛЛЕЛЬНО, затем интегрируй и верифицируй (`just check`).
- Read-only исследование/аудит → запускай широко (риска нет). Параллельные ПРАВКИ → строгие границы
  файлов на агента; общие файлы (`models.py`, `packages/shared`, `components/ui`) — read-only, в отчёт.
  Коммит и `just check` делает ТОЛЬКО оркестратор (единый git-индекс/гейт).

## Железные правила

- **TDD на pure-логике** (тайминги субтитров, crop 9:16, группировка слов, квота/биллинг, пересечения
  сегментов, маппинг офсетов): сначала падающий тест, потом код.
- **Гейт перед КАЖДЫМ коммитом — `just check` зелёный** (ruff + mypy + tsc + eslint + unit-тесты +
  anti-drift). Conventional commits. Коммит ТОЛЬКО из PowerShell (pre-commit хук зовёт `just`). Если
  ruff-format переформатировал → `git add` + повторный коммит.
- **Типы — только codegen** из `services/worker/app/models.py` → `just types`. Руками `packages/shared/*`
  НЕ трогать. Контракты меняются только в `models.py`.
- **Никаких тихих фолбэков** (`except: pass`, молчаливый default): ошибка → `JobError` / статус failed /
  явный лог.
- **Верь, но проверяй:** «готово/работает» — только с РЕАЛЬНЫМ выводом проверки (команда + результат).
  Баг/непонятка → скилл `systematic-debugging` (корень до фикса, не угадывай). Не сходится 2× → СТОП, спроси.
- **Без скоупа сверх задачи** — не тащи лишние правки/рефакторы.
- **Не трогай репозиторий лендинга `Shorts-Automatizator`** (правило #10).
- **Фронт закладывает задержки бэка:** polling статуса, скелетоны, таймаут+ретрай, понятные состояния —
  пайплайн долгий (минуты).
- **Reframe (кадрирование fill↔fit↔split) — читай ЭТО перед любой правкой `stage3_reframe` /
  `stage5_render` / `editor/reframe_cache` / `main.py:get_clip_reframe`** (детали:
  `docs/REFRAME_FPS_GRID_INVARIANT.md`). Уроки сессии 2026-06-23 (стоили дней — НЕ повторять):
  1. **ДВА СЛОЯ.** Код считает план, но редактор/превью читают ЗАПЕЧЁННЫЕ регионы из
     `job_artifacts.reframe_regions` (БД) через `/reframe` fast-path БЕЗ пересчёта. → Любая правка
     wide/tight-логики НЕВИДИМА на готовых клипах, пока не сбросишь
     `update public.job_artifacts set reframe_regions = null;` (ПОСЛЕ деплоя). Меняешь reframe-логику →
     ОБЯЗАТЕЛЬНО чисти кэш регионов, иначе «фикс не работает».
  2. **«Флеш» = переход ВЕРТИКАЛЬНЫЙ(fill)↔ГОРИЗОНТАЛЬНЫЙ(fit), НЕ пан внутри fill.** Рендер — жёсткий
     cut (мгновенно); он ДОЛЖЕН попадать РОВНО на контент-склейку, тогда смена режима маскируется.
  3. **`-1` в `detect_scene_cuts` fps-ЗАВИСИМ** (`_scene_cut_offset`): дробный fps (29.97/23.976) → −1,
     ЦЕЛЫЙ (24/25/30/60) → 0. На 25fps старый жёсткий −1 уводил границу на кадр РАНЬШЕ склейки = 1 кадр
     старого шота в НОВОМ режиме = флеш. (Т.е. флеш ломался ИМЕННО на 25fps — прежнее «ломается только
     на ≠25fps» было НЕВЕРНЫМ.)
  4. **Ложный wide = спурьез-трек.** 2-shot ставим по ≥2 лицам с покрытием ≥ coverage_min; отражения/
     IOU-разрывы дают трек с покрытием 0–6% → coverage-гейт в `_is_wide_shot` их режет. Низкий порог
     размаха / min_hold → ложные wide + строб; не крути их «на глаз».
  5. **Превью ≠ рендер — ДВЕ ловушки времени/анимации.** (а) Время/режим в `PreviewPlayer` ОБЯЗАТЕЛЬНО
     гнать через `requestVideoFrameCallback` (покадрово), НЕ через `timeupdate` (~4Hz) — иначе режим
     ОТСТАЁТ от кадра до ~250мс → новый шот в старом кропе = мельк, даже на СВЕЖЕМ аплоаде (это не
     граница/кэш!). (б) Мастер-видео НЕ должно CSS-анимировать смену режима (`object-position`
     transition) — «зум-эффект» на стыке; switch мгновенный. Рендер при этом кадрово-ТОЧЕН (0 лага) →
     скачанный клип чист ВСЕГДА; превью — живое приближение (~1 кадр React-латентности).
  6. **ПРОВЕРЯТЬ ТОЛЬКО ГЛАЗАМИ, ПОКАДРОВО, на РЕАЛЬНОМ видео-рендере на стыке.** Unit-тесты и
     `verify_grid_fix.py` (Δ=0) НЕ ловят 1-кадровый рассинхрон с КОНТЕНТ-склейкой (только grid-
     целочисленность). Локальный файл ≠ загруженный R2-source (разные треки) → боевой `/reframe` = истина.
- **Экономика первоклассно:** cost/latency на стадию → `runs.jsonl`; сравнение моделей / юнит-экономика →
  `docs/BENCHMARKS.md` (живая таблица).
- **Язык интерфейса — RU/EN тумблер ЖИВОЙ в core-фанеле (2026-06-30, обновляет правило 2026-06-18).**
  Раньше был курс «только английский»; теперь основатель ОДОБРИЛ двуязычие и оно ОТГРУЖЕНО для основного
  фанела: **лендинг `/` → auth → дашборд → загрузка** двуязычны. Стек: **`next-intl` в COOKIE-режиме**
  (БЕЗ `[locale]`-сегмента и БЕЗ `proxy.ts`): локаль читается из cookie `NEXT_LOCALE` в `i18n/request.ts`
  (`getRequestConfig`); провайдер монтируется один раз в `app/layout.tsx` (+ динамический `<html lang>`).
  Каталоги auth/app — `apps/web/messages/{en,ru}.json` (через `t()` / `useTranslations` / `getTranslations`).
  Копия лендинга — двуязычный `Record<Locale,…>` в `apps/web/lib/landingContent.ts` (как `lib/useCases.ts`),
  резолв через `getLandingContent(locale)`. Тумблер `EN / RU` — `components/i18n/LocaleSwitcher.tsx` (пишет
  cookie + `router.refresh()`), стоит в лендинг-Nav, App-header и лендинг-Footer.
  - **Правило для НОВЫХ строк:** НИКОГДА не хардкодь user-facing текст — добавляй ключ СРАЗУ в ОБА
    каталога `en.json` + `ru.json` (иначе tsc/рантайм упадёт на отсутствующем ключе) и зови через `t()`;
    для лендинга — добавляй в `en` + `ru` блоки `landingContent.ts`. Русский должен быть НАСТОЯЩИМ
    (идиоматичным), не машинным; никакого смешанного RU/EN в одной строке.
  - **ЕЩЁ английский (отложено):** редактор (`components/editor/**`, `ExportMenu.tsx`) и worker-side
    `JobError`-reason'ы (`services/worker/**`). Эти строки пока английские; их перевод — следующие фазы.

## Границы кода

- `pipeline/*` — чистые функции (вход→выход, без HTTP/DB). Склейка — только в `tasks.py` / `run.py`.
  Дублируется логика → выноси в ОДНУ функцию. Docstring на публичных функциях.

## Грабли инструментов (повторяется каждую сессию — критично)

- **PATH refresh в КАЖДОМ PowerShell-вызове** (иначе нет `just`/`uv`/`ffmpeg`):
  `$env:PATH = [Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [Environment]::GetEnvironmentVariable("Path","User")`
- **Bash-инструмент ≠ PowerShell:** в Bash НЕТ `ffmpeg`/`just`/`uv` (winget-PATH). Любые прогоны
  пайплайна/ffmpeg/`just`/`uv` — через PowerShell с PATH refresh.
- **PowerShell держит cwd между вызовами** → всегда абсолютные пути или `Set-Location` в начале.
- **Коммит-сообщение** с кириллицей: писать в файл (UTF-8 без BOM) + `git commit -F <файл>` (пайп бьёт
  кодировку в `?????`).
- **`modal deploy` с Windows:** сначала `$env:PYTHONIOENCODING="utf-8"` (иначе CLI падает на `✓` charmap).
  После правки кода воркера/биллинга → `modal deploy deploy/modal/worker.py`.
- **После правки кода воркера локально — перезапусти `uvicorn`** (запущен без `--reload`).
